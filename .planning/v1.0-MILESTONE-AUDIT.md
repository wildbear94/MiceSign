## Critical Integration Issues

### 1. USER role cannot select approvers (APR-01) — CRITICAL

**Files:**
- `frontend/src/features/approval/components/OrgTreePickerModal.tsx:60`
- `frontend/src/features/admin/api/departmentApi.ts:14`
- `backend/.../controller/OrganizationController.java` (correct endpoint, unused)

**Impact:** Any USER-role employee opening the ApprovalLineEditor gets 403. Org tree is empty. Document submission is blocked for regular users. This breaks the core approval workflow.

**Fix:** Change OrgTreePickerModal to call `/api/v1/organization/departments` instead of `/api/v1/admin/departments`.

### 2. PendingApprovalResponse type mismatch (DASH-01) — HIGH

**Files:**
- `frontend/src/features/approval/types/approval.ts:31-41`
- `frontend/src/features/approval/pages/PendingApprovalsPage.tsx:104,109`
- `frontend/src/features/dashboard/components/PendingList.tsx:95`

**Impact:** Date column renders "Invalid Date", department column blank on pending approvals list and dashboard.

**Fix:** Align frontend type to match backend: `submittedAt`→`createdAt`, `drafterDepartmentName`→`departmentName`.

### 3. rewriteDocument() missing audit log (AUD-01) — MEDIUM

**File:** `backend/.../service/DocumentService.java:396-444`

**Impact:** 재기안 (resubmission) creates a new document without audit trail entry.

**Fix:** Add `auditLogService.log()` call for DOC_REWRITE action.

### 4. getCompletedApprovals() dead code — MEDIUM

**File:** `backend/.../service/ApprovalService.java:197-213`

**Impact:** "완료된 문서" page returns drafter's docs instead of docs the user approved. `actedLines` variable built but never used.

**Fix:** Use `actedLines` to query documents the user has approved/rejected.
