## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `gray-800` (dark) | Condition editor background, input backgrounds |
| Secondary (30%) | `gray-50` / `gray-700` (dark) | Collapsible section header bg, preview field bg |
| Accent (10%) | `blue-600` / `blue-400` (dark) | "조건 추가" button text, focus rings on rule inputs, reset button |
| Destructive | `red-500` / `red-400` (dark) | Rule delete button hover, required asterisk (*), validation errors |
| Condition badge | `amber-100`/`amber-700` / `amber-900/40`/`amber-300` (dark) | Condition icon badge on FieldCard header (D-06) |

Accent reserved for:
- "조건 추가" add button text (`text-blue-600`)
- Focus ring on rule dropdowns/inputs (`focus:ring-blue-600`)
- "초기화" reset button in preview panel (`text-blue-600`)
- NOT used on: section collapse toggles, delete buttons (these use `gray-*` or `red-*` respectively)

Source: Existing FieldConfigEditor and Phase 23 color patterns.

## Layout Contract

### Condition Rule Section (inside FieldCard, below FieldConfigEditor content)

```
+----------------------------------------------------------+
| [ChevronRight/Down] 조건 규칙  [amber badge: Zap icon]   |  <- collapsible header (D-01, D-06)
+----------------------------------------------------------+
|                                                          |
|  (When collapsed: hidden)                                |
|  (When expanded, no rule -- D-03):                       |
|                                                          |
|  조건 규칙이 없습니다                                      |
|  [+ 조건 추가]                                            |
|                                                          |
|  (When expanded, rule exists -- D-07):                   |
|                                                          |
|  IF  [필드 선택 ▼]  [연산자 ▼]  [값 입력/선택]             |
|  THEN  [액션 선택 ▼]                         [삭제]       |
|                                                          |
+----------------------------------------------------------+
```

### Rule Editor Row Detail (D-07)

The IF-THEN layout uses a stacked two-row approach for clarity within the narrow FieldConfigEditor width:

```
+----------------------------------------------------------+
| IF                                                       |
| [소스 필드 드롭다운 ▼        ]                             |  <- full width
| [연산자 드롭다운 ▼]  [값 입력 필드/드롭다운       ]         |  <- grid-cols-2 gap-3
|                                                          |
| THEN                                                     |
| [액션 드롭다운 ▼             ]           [Trash2 삭제]     |  <- flex justify-between
+----------------------------------------------------------+
```

Grid layout: `grid grid-cols-2 gap-3` for operator + value row. Full width for source field dropdown and action row.

### In/NotIn Checkbox List (D-11)

When operator is `in` or `notIn` and source is `select` type:

```
+----------------------------------------------------------+
| [x] 옵션 A                                               |
| [x] 옵션 B                                               |
| [ ] 옵션 C                                               |
| [ ] 옵션 D                                               |
+----------------------------------------------------------+
```

Container: `max-h-40 overflow-y-auto border rounded-lg p-2 space-y-1`

### FieldCard Header Badge (D-06)

```
+----------------------------------------------------------+
| [::] [ChevronRight] [TypeBadge] 필드 라벨  [Zap] [삭제]  |
+----------------------------------------------------------+
                                              ^^^^^
                                    amber badge, only when rule exists
```

Badge: `Zap` icon from Lucide, `w-3.5 h-3.5`, wrapped in `bg-amber-100 text-amber-700 dark:bg-amber-900/40 dark:text-amber-300 rounded px-1 py-0.5` pill.

### Interactive Preview (D-16, D-17, D-19, D-20)

```
+----------------------------------------------------------+
| 미리보기                                  [초기화] [전체]  |  <- header with reset button
+----------------------------------------------------------+
| [양식 제목 입력 필드         ]                              |
|                                                          |
| 필드 1 라벨                                               |
| [입력 가능 필드             ]                              |
|                                                          |
| 필드 2 라벨 *               <- required by condition      |
| [입력 가능 필드             ]                              |
|                                                          |
| (필드 3: hidden by condition -- not rendered)             |
|                                                          |
+----------------------------------------------------------+
```

- All fields render enabled inputs (D-16)
- Hidden fields are completely removed from DOM, not display:none (D-17, no animation)
- Required fields show red `*` next to label (D-18)
- Initial load shows all fields (D-19)
- "초기화" button resets all formValues to empty (D-20)

## Keyboard & Accessibility

| Interaction | Key | Behavior |
|-------------|-----|----------|
| Navigate between dropdowns | Tab | Standard tab order: source field -> operator -> value -> action |
| Open dropdown | Space / Enter | Standard native select behavior |
| Delete rule | Click Trash2 button | Immediate removal (D-14) |
| Expand/collapse condition section | Click header or Enter when focused | Toggle expanded state |
| Checkbox list (in/notIn) | Space | Toggle individual checkbox |
| Preview reset | Click or Enter | Clear all formValues |

All dropdown elements must use native `<select>` elements or have `aria-label` attributes.
Condition section header must have `role="button"` and `aria-expanded` attribute.
Checkbox list items must have associated `<label>` elements.

## Validation Contract

| Rule | Trigger | Visual Feedback |
|------|---------|-----------------|
| Rule references nonexistent field | Save button click (D-26) | Error toast: `templates.condition.validationError` + save blocked (D-27) |
| Rule has invalid operator for source type | Save button click | Error toast + save blocked |
| Rule has empty required fields (no source, no operator, no action) | Save button click | Error toast + save blocked |
| Self-reference prevention | Source field dropdown render (D-12) | Current field excluded from dropdown options |
| Circular reference prevention | Source field dropdown render (D-13) | Fields that would create cycle excluded from dropdown options |
| Source field type restriction | Source field dropdown render (D-05) | Only text/number/date/select fields shown |
| Target field type restriction | Condition section visibility (D-04) | Condition section not rendered for staticText/hidden field types |
