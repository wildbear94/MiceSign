**2. NotificationService.java -- event listener + recipient resolution + retry (per D-01, D-02, D-04, D-10):**
Create `backend/src/main/java/com/micesign/service/NotificationService.java`:
- `@Service` with constructor injection of `EmailService`, `NotificationLogRepository`, `ApprovalLineRepository`, `UserRepository`
- `@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)` + `@Async` on `handleNotificationEvent(ApprovalNotificationEvent event)`:
  - Call `resolveRecipients(event)` to get `List<User>`
  - For each recipient, call `sendWithRetry(recipient, event)`

- **Recipient resolution (per D-04, RESEARCH Pitfall 6 table):**
  - `resolveRecipients(ApprovalNotificationEvent event)` returns `List<User>`:
    - SUBMIT: all approval line members with type APPROVE or AGREE (NOT REFERENCE), fetched via `approvalLineRepository.findByDocumentIdOrderByStepOrderAsc(event.getDocument().getId())`, map to `line.getApprover()`
    - APPROVE intermediate (document status still SUBMITTED): next pending approver only -- find first PENDING line with stepOrder > currentStep and type APPROVE or AGREE
    - APPROVE final (document status is APPROVED): drafter only -- `event.getDocument().getDrafter()`
    - REJECT: drafter only -- `event.getDocument().getDrafter()`
    - WITHDRAW: all approval line members (APPROVE + AGREE types) -- same query as SUBMIT
  - Distinguish APPROVE intermediate vs final by checking `event.getDocument().getStatus()`: if `APPROVED` then final, else intermediate

- **sendWithRetry (per D-10):**
  - Max retries = 2, delays = `{1000L, 3000L}`
  - Create `NotificationLog` with PENDING status, save to DB
  - Loop `attempt = 0` to `maxRetries`:
    - Try: call `emailService.sendEmail(recipient.getEmail(), subject, templateName, variables)`
    - On success: set log status = SUCCESS, sentAt = now, retryCount = attempt, save, return
    - On failure: if more retries, set log status = RETRY, retryCount = attempt + 1, save, `Thread.sleep(delays[attempt])`; if last attempt, set status = FAILED, errorMessage = `e.getMessage()` (truncate to 1000 chars), save
  - Entire method wrapped in try-catch to never propagate exceptions (same non-blocking pattern as AuditLogService)
  - Use SLF4J logger for warnings on failure
