# Phase 9 — UI Design Contract

> Visual and interaction contract for SMTP Email Notifications frontend. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** Phase 9 is primarily backend (event-driven email delivery via Spring ApplicationEvent + @Async). The frontend scope is limited to a single SUPER_ADMIN notification history management page (D-12, D-13 from CONTEXT.md) and email HTML templates rendered by Thymeleaf on the backend. This contract covers both.

## Layout Contract

### NotificationLogPage Layout

```
+----------------------------------------------------------+
| 알림 이력                                                  |  <- Heading, text-xl
+----------------------------------------------------------+
| [상태 ▼]  [이벤트 타입 ▼]  [시작일]  [종료일]  [검색]        |  <- Filter bar
+----------------------------------------------------------+
| [Table: 수신자, 이벤트, 문서, 상태, 재시도, 발송일시, 액션]    |  <- Paginated table
| ...                                                       |
+----------------------------------------------------------+
| [Pagination]                                              |
+----------------------------------------------------------+
```

- Filter bar: `flex flex-wrap gap-4 mb-6` (same pattern as AuditLogFilters)
- Status select: options are "전체", "성공", "실패", "대기"
- Event type select: options are "전체", "제출", "승인", "반려", "회수"
- Date range: two `<input type="date">` fields
- Search button: `bg-blue-600 text-white px-4 py-2 rounded-lg text-sm font-medium`

### NotificationLogTable Columns

| Column | Width | Content |
|--------|-------|---------|
| 수신자 | auto | Recipient user name |
| 이벤트 | 80px | Event type label (제출/승인/반려/회수) |
| 문서 | auto | Document title (truncated at 30 chars) + document number |
| 상태 | 80px | NotificationStatusBadge (SUCCESS/FAILED/PENDING) |
| 재시도 | 60px | retry_count number (0, 1, 2) |
| 발송일시 | 140px | YYYY-MM-DD HH:mm:ss format |
| 액션 | 80px | "재발송" button (only visible on FAILED rows) |

### Resend Button

- Visible only on rows with status FAILED
- Style: `text-blue-600 hover:text-blue-700 dark:text-blue-400 text-sm font-medium` (text button, not filled)
- Click triggers ConfirmDialog before API call
- During resend request: button text changes to "발송 중..." with `opacity-50 pointer-events-none`
- After success: row status updates to SUCCESS (via TanStack Query invalidation)
- After failure: toast or inline error message

## Interaction Contract

### Notification Log Interactions

| Interaction | Behavior |
|-------------|----------|
| Change filter + click 검색 | Refetch notification logs with new filter parameters, reset to page 0 |
| Press Enter in filter input | Same as clicking 검색 |
| Click pagination | Load page N of filtered results |
| Click 재발송 on FAILED row | Show ConfirmDialog; on confirm, POST resend API; on success, invalidate query |
| No row click navigation | Notification log rows are not clickable for navigation |

### Loading States

| Component | Loading Behavior |
|-----------|-----------------|
| Notification log table | Show 3 skeleton rows (`h-12 bg-gray-200 dark:bg-gray-700 rounded animate-pulse`), matching AuditLogTable pattern |
| Resend button | Disabled + "발송 중..." text during API call |

### Error States

| Component | Error Behavior |
|-----------|---------------|
| Table data fetch | Show error banner above table: "데이터를 불러오는 데 실패했습니다. 잠시 후 다시 시도해 주세요." |
| Resend API failure | Toast notification: "재발송에 실패했습니다. 잠시 후 다시 시도해 주세요." |

## Accessibility Notes

| Element | Requirement |
|---------|-------------|
| Filter controls | Associated `<label>` elements (sr-only) for each input, matching AuditLogFilters pattern |

## Dark Mode

All components must support dark mode via Tailwind `dark:` prefix, following established patterns:
- Backgrounds: `bg-white dark:bg-gray-900` (page), `bg-gray-50 dark:bg-gray-800` (table headers)
- Borders: `border-gray-200 dark:border-gray-700`
- Text: `text-gray-900 dark:text-gray-50` (heading), `text-gray-600 dark:text-gray-400` (body)
- Inputs: `bg-white dark:bg-gray-800 text-gray-900 dark:text-gray-50 border-gray-300 dark:border-gray-600`
