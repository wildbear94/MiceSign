    ## 2. Mutation 실시간 갱신 (DASH-05)

    **사전**: 대시보드 탭을 띄워둔 채 별도 탭 또는 같은 탭에서 액션 수행 후 대시보드로 복귀.

    ### 2.2 반려 (useReject)
    - [ ] 결재 반려 수행
    - [ ] "결재 대기" `-1`, "반려" `+1` (기안자 시점에서)

    - [ ] "진행 중" 카운트 `+1` (본인 기안자 시점)

    ### 2.5 갱신 지연 없음
    - [ ] 각 mutation 후 최대 2초 이내 카운트 변화가 반영됨
