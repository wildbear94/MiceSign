## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-T3 (cache pollution) | Information Disclosure | `['dashboard', 'summary']` queryKey | mitigate | queryKey 가 role-independent 이지만 queryFn 이 axios 를 통해 현재 인증 토큰으로만 호출되므로 BE 가 본인 스코프 응답만 반환. logout 시 `queryClient.clear()` 호출 필요 — 해당 호출은 기존 authStore.clearAuth 동작 또는 App 레벨 logout handler 에 존재 여부 확인 (Plan 05 follow-up). 본 Plan 에서는 queryKey 변경만 담당 |
| T-31-T3-b | Information Disclosure | placeholderData leak across role switch | accept | 로그인 전환 시 Zustand clearAuth → QueryClient.clear() 패턴이 없으면 이전 사용자 데이터가 placeholderData 로 잠시 노출될 수 있음. 단 Vite dev server 에서 로그인 전환은 full page refresh 를 수반 (authStore + routing), placeholderData 는 페이지 세션 내에서만 유효 → 위험도 낮음. follow-up 으로 Plan 05 UAT 에서 logout → login 시 대시보드 깜빡임 수동 확인 |
