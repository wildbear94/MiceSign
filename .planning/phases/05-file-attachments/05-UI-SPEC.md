## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, progress bar height |
| sm | 8px | Compact element spacing, file item internal gaps |
| md | 16px | Default element spacing, drop zone internal padding |
| lg | 24px | Section padding, file attachment area padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Drop zone vertical padding (center content) |
| 3xl | 64px | Not used this phase |

Exceptions:

| Value | Justification |
|-------|---------------|
| 44px (`h-11`) | Touch target minimum per WCAG 2.5.8. Used for click-to-browse button inside drop zone. |

Source: Identical to Phase 4 spacing scale. Drop zone uses `p-6` (24px) and `py-12` (48px) for centered content layout.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `gray-900` (dark) | Page background, main content area |
| Secondary (30%) | `gray-50` / `gray-800` (dark) | File item backgrounds, drop zone background on idle |
| Accent (10%) | `blue-600` | Drop zone active border, upload progress bar fill, browse button |
| Destructive | `red-600` | Upload error text, failed file icon |

Accent reserved for:
- Drop zone border and background when dragging over (`border-blue-500 bg-blue-50`)
- Upload progress bar fill (`bg-blue-600`)
- "파일 찾기" browse link text inside drop zone (`text-blue-600`)
- File usage status count when at limit (`text-blue-600` for emphasis)

Additional semantic colors for upload states:

| State | Color | Usage |
|-------|-------|-------|
| Uploading | `blue-600` | Progress bar fill, percentage text |
| Success | `green-600` | Completed upload checkmark icon |
| Error | `red-600` | Failed upload error text and icon |

Source: Accent `blue-600` and destructive `red-600` carried from Phase 4. Upload state colors follow established status badge pattern (green for success, red for error).

## Interaction Contracts

### File Selection & Validation (D-01, D-03, D-04)

| Property | Value |
|----------|-------|
| Trigger (drag) | Drop files onto FileDropZone |
| Trigger (browse) | Click drop zone or "파일 찾기" text |
| Multi-select | `<input type="file" multiple>` allows multiple files |
| Cumulative | New files add to existing list, never replace (D-03) |
| Validation timing | Immediate at selection time, before upload starts (D-04) |
| Invalid file handling | Rejected files are never added to list; inline error shown below drop zone for 5 seconds |
| Valid file handling | Added to file list immediately with "pending" state, upload starts automatically |

### Upload Progress (D-02)

| Property | Value |
|----------|-------|
| Trigger | Automatic after file passes validation |
| Progress display | Individual progress bar per file (D-02) |
| Progress source | Axios `onUploadProgress` callback |
| States | pending (0%) -> uploading (1-99%) -> complete (100%) -> confirmed (server response) |
| Failure | Progress bar turns red; error text replaces percentage; file icon changes to `AlertCircle` |
| Retry | No automatic retry on frontend. User can delete failed file and re-add. |
| Concurrent uploads | Files upload one at a time (sequential) to avoid overwhelming the server |

### File Delete (D-09)

| Property | Value |
|----------|-------|
| Trigger | Click X button on file item |
| Confirmation | None for draft documents (D-09) |
| Effect | Immediate removal from list + API call to backend to delete from Drive |
| Submitted documents | No delete button rendered (immutability rule) |
| During upload | Cancel ongoing upload (abort Axios request) + remove from list |

### File Download (D-10, D-11, D-12)

| Property | Value |
|----------|-------|
| Trigger | Click "다운로드" text button on file item |
| Method | Programmatic download via Axios GET with `responseType: 'blob'`, then create temporary `<a>` link |
| Filename | Use `Content-Disposition` header from backend response |
| Authorization | Backend proxy validates document-level access; frontend hides button for unauthorized users (D-11) |
| Bulk download | Not supported in MVP (D-12) |
| Loading state | "다운로드" text changes to spinning `Loader2` icon during download |

### Drop Zone State Machine

```
IDLE ──(dragenter)──> DRAG_OVER ──(drop)──> VALIDATING ──> IDLE
  ^                       |                       |
  |                       └──(dragleave)──────────┘
  |
  └──(click)──> FILE_PICKER ──(select)──> VALIDATING ──> IDLE
```

| State | Visual |
|-------|--------|
| IDLE | Dashed gray border, gray upload icon, default text |
| DRAG_OVER | Dashed blue border, blue-50 background, "놓으세요" text |
| DISABLED | 50% opacity, not-allowed cursor (10 files reached or 200MB reached) |

## Accessibility Notes

All icon-only buttons in this phase must include an `aria-label` attribute:

| Component | Button | aria-label value |
|-----------|--------|-----------------|
| `FileItem` | Delete (X icon) | "파일 삭제" |
| `FileDropZone` | (entire zone is clickable) | `role="button" aria-label="파일 첨부 영역. 클릭하여 파일을 선택하세요"` |

Additional accessibility requirements:

| Requirement | Implementation |
|-------------|----------------|
| Drop zone keyboard access | `tabIndex={0}`, `onKeyDown` for Enter/Space to open file picker |
| Progress announcement | `aria-live="polite"` on upload progress region |
| Error announcement | `aria-live="assertive"` on validation error messages |
| File list | `role="list"` on container, `role="listitem"` on each FileItem |
