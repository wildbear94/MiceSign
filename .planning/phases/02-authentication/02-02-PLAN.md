    **3. Create PasswordService** (`service/PasswordService.java`):
    - `@Service @Transactional` with injected UserRepository, RefreshTokenRepository, PasswordEncoder

    - **validatePasswordStrength(String password)**: per D-27, returns true if:
      - Length >= 8
      - Contains at least 2 of: uppercase letter, lowercase letter, digit, special character (`!@#$%^&*()_+-=[]{}|;:',.<>?/~`)
      - Return false with specific error message if validation fails

    - **changePassword(Long userId, PasswordChangeRequest request, String currentTokenHash)**:
      1. Validate `newPassword.equals(confirmPassword)` — error code AUTH_PASSWORD_MISMATCH if not
      2. Find user by ID, verify `passwordEncoder.matches(request.currentPassword(), user.getPassword())` — error AUTH_WRONG_PASSWORD if not
      3. Validate password strength — error AUTH_WEAK_PASSWORD if fails
      4. Set `user.setPassword(passwordEncoder.encode(request.newPassword()))`
      5. Set `user.setMustChangePassword(false)` — clears force-change flag per D-30
      6. Invalidate all OTHER refresh tokens per D-32: find all tokens by userId, delete all except the one matching currentTokenHash
      7. Save user
      8. Return success

    - **adminResetPassword(Long adminUserId, Long targetUserId, AdminPasswordResetRequest request)**:
      1. Load admin user, verify role is ADMIN or SUPER_ADMIN
      2. Load target user. If target is SUPER_ADMIN and admin is ADMIN (not SUPER_ADMIN), return 403 AUTH_FORBIDDEN — ADMIN cannot reset SUPER_ADMIN
      3. Validate password strength on newPassword
      4. Set `target.setPassword(passwordEncoder.encode(request.newPassword()))`
      5. Set `target.setMustChangePassword(true)` per D-29/D-30
      6. Invalidate ALL refresh tokens for target user (`refreshTokenRepository.deleteByUserId(targetUserId)`) — force re-login
      7. Save target user

    - **unlockAccount(Long adminUserId, Long targetUserId)**:
      1. Verify admin role (ADMIN or SUPER_ADMIN)
      2. Load target user, set `failedLoginCount = 0`, `lockedUntil = null` per D-38
      3. Save user

    **4. Create PasswordController** (`controller/PasswordController.java`):
    
    - Password change endpoint at auth controller level:
      `@PutMapping` on `@RequestMapping("/api/v1/auth")` class or add to existing AuthController — decide based on how AuthController is structured in Plan 01. If separate class: `@RestController @RequestMapping("/api/v1/auth")`

    - `@PutMapping("/password")`: accepts `@Valid @RequestBody PasswordChangeRequest`. Extract userId from SecurityContext (`CustomUserDetails`). Extract current refresh token hash from cookie (need `@CookieValue("refreshToken")` and hash it). Call `passwordService.changePassword(userId, request, currentTokenHash)`. Return `ApiResponse.ok("비밀번호가 변경되었습니다.")` per D-35.

    - Admin endpoints:
      `@RestController @RequestMapping("/api/v1/admin/users")`
    
    - `@PostMapping("/{id}/reset-password")`: accepts path var userId and `@Valid @RequestBody AdminPasswordResetRequest`. Extract admin userId from SecurityContext. Call `passwordService.adminResetPassword(adminId, targetUserId, request)`. Return `ApiResponse.ok("비밀번호가 초기화되었습니다.")`.

    - `@PostMapping("/{id}/unlock")`: Extract admin userId from SecurityContext. Call `passwordService.unlockAccount(adminId, targetUserId)`. Return `ApiResponse.ok("계정 잠금이 해제되었습니다.")` per D-38.

    Note: The password change endpoint path is `/api/v1/auth/password` per RESEARCH.md API summary. The admin endpoints are under `/api/v1/admin/users/{id}/` which requires ADMIN or SUPER_ADMIN role (already configured in SecurityConfig).
