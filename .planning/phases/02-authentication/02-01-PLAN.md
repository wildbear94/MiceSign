    - `@Service @Transactional` with injected UserRepository, RefreshTokenRepository, PasswordEncoder, JwtTokenProvider
    - `@Value("${jwt.refresh-token-ttl}")` for refresh TTL

    - **login(LoginRequest request, String deviceInfo)**:
      1. Find user by email (`userRepository.findByEmail`), throw if not found with AUTH_INVALID_CREDENTIALS
      2. Check if account locked: if `user.getLockedUntil() != null && user.getLockedUntil().isAfter(LocalDateTime.now())`, return error AUTH_ACCOUNT_LOCKED with lockedUntil timestamp in response (per D-36, D-42)
      3. If lockout expired (lockedUntil is in the past), reset: `user.setFailedLoginCount(0); user.setLockedUntil(null);` per D-39
      4. Verify password with `passwordEncoder.matches(request.password(), user.getPassword())`
      5. On password mismatch: increment `failedLoginCount`. If count >= 5, set `lockedUntil = LocalDateTime.now().plusMinutes(15)` per D-04/AUTH-04. Save user. Return AUTH_INVALID_CREDENTIALS with remainingAttempts = 5 - failedLoginCount per D-37
      6. On success: reset `failedLoginCount` to 0, `lockedUntil` to null per D-39. Generate access token via JwtTokenProvider. Generate UUID refresh token, hash with SHA-256, store in RefreshToken entity (userId, tokenHash, deviceInfo, expiresAt = now + refreshTtl). Save user and refresh token. Return LoginResponse with accessToken and UserProfileDto.
      7. Determine cookie maxAge: if `request.rememberMe()` is true, 14 days; if false, omit maxAge (session cookie) per D-02 and Pitfall 2

    - **refresh(String rawRefreshToken)**:
      1. Hash incoming token with SHA-256
      2. Look up in DB by tokenHash. If not found, return error (possible token reuse — per D-18, invalidate ALL tokens for that user if suspicious)
      3. Check expiresAt. If expired, delete token and return error
      4. Token rotation per D-18: delete old token, generate new UUID, hash and store new RefreshToken with same userId and new expiresAt
      5. Load user from DB, generate new accessToken
      6. Return RefreshResponse with new accessToken and UserProfileDto per D-15

    - **logout(String rawRefreshToken)**:
      1. Hash incoming token, find in DB, delete per D-24 (only current session's token)
      2. Return success

    - Private helper `hashToken(String rawToken)`: SHA-256 hash, Base64 encode (same as RESEARCH.md code example)
    - Private helper `buildUserProfile(User user)`: maps User entity to UserProfileDto
