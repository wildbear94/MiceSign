    ## §1. MariaDB slow query log 활성화 (1회 작업)

    ### 1.1 임시 활성화 (재시작 시 초기화)
    \`\`\`sql
    SET GLOBAL slow_query_log = ON;
    SET GLOBAL long_query_time = 1.0;
    SET GLOBAL slow_query_log_file = '/var/log/mysql/slow.log';
    SET GLOBAL log_output = 'FILE,TABLE';   -- 파일 + mysql.slow_log 둘 다
    \`\`\`

    ### 1.2 영구화 (my.cnf)
    `/etc/mysql/mariadb.conf.d/50-server.cnf` 의 `[mysqld]` 섹션:
    \`\`\`ini
    slow_query_log = 1
    long_query_time = 1.0
    slow_query_log_file = /var/log/mysql/slow.log
    log_output = FILE,TABLE
    \`\`\`

    적용:
    \`\`\`bash
    sudo systemctl restart mariadb
    mysql -uroot -e "SHOW VARIABLES LIKE 'slow_query%';"
    \`\`\`

    기대 출력에 `slow_query_log=ON` + `slow_query_log_file=/var/log/mysql/slow.log` 확인.

    ### 1.3 로그 파일 권한 + rotation
    \`\`\`bash
    sudo touch /var/log/mysql/slow.log
    sudo chown mysql:mysql /var/log/mysql/slow.log
    sudo chmod 640 /var/log/mysql/slow.log
    \`\`\`

    `/etc/logrotate.d/mariadb-slow` (로그 무한 증가 방지):
    \`\`\`
    /var/log/mysql/slow.log {
        weekly
        rotate 12
        compress
        delaycompress
        missingok
        notifempty
        sharedscripts
        postrotate
            mysqladmin -uroot flush-logs
        endscript
    }
    \`\`\`

    ## §2. 3 신호 정의 — D-S2

    아래 **3 신호 중 하나라도** 발동 시 §3 재실측 절차로.

    ### 신호 (a) — 사용자 체감 검색 지연 신고
    - 트리거: 내부 사용자 1명 이상이 "문서 검색이 느리다" 또는 "결과가 안 나온다" 신고.
    - 채널: 사내 메신저 / 이메일 / GSD issue tracker.
    - 운영자 즉시 §3 절차 + 신고자에게 ack 회신.

    ### 신호 (b) — slow query log 의 검색 쿼리 1초 초과
    매주 월요일 (또는 일일) 점검:

    \`\`\`sql
    -- 지난 7일 slow log 중 documents/search 백엔드 쿼리 (Hibernate 가 발급하는 SELECT)
    SELECT
      DATE(start_time) AS day,
      COUNT(*) AS slow_count,
      MAX(query_time) AS max_query_time
    FROM mysql.slow_log
    WHERE start_time >= DATE_SUB(NOW(), INTERVAL 7 DAY)
      AND sql_text LIKE '%FROM document%'
      AND sql_text LIKE '%approval_line%'
    GROUP BY DATE(start_time)
    ORDER BY day DESC;
    \`\`\`

    `slow_count > 0` 한 row 라도 발견 시 §3 절차.

    파일 기반 확인 (옵션):
    \`\`\`bash
    sudo grep -A 5 'FROM document' /var/log/mysql/slow.log | grep 'Query_time:' | sort -k 2 -n -r | head -20
    \`\`\`

    ### 신호 (c) — 동시 활성 사용자 30명 초과
    \`\`\`sql
    -- 직전 5분간 활동한 user (audit_log 기반 근사치)
    SELECT COUNT(DISTINCT user_id) AS active_users
    FROM audit_log
    WHERE created_at >= DATE_SUB(NOW(), INTERVAL 5 MINUTE);
    \`\`\`

    `active_users >= 30` 도달 시 §3 절차 (사전 예방적 재실측).

    크론으로 자동화 옵션 (선택):
    \`\`\`bash
    # /etc/cron.d/micesign-active-users
    */5 * * * * mysql micesign root /opt/micesign/scripts/check-active-users.sh
    \`\`\`
    (스크립트 작성은 본 문서 범위 밖 — 발동 시점에 추가)
