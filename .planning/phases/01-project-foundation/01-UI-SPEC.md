# Phase 1 — UI Design Contract

> Visual and interaction contract for the project foundation phase. This phase delivers no user-facing screens — its UI deliverable is the TailwindCSS configuration and design token foundation that all subsequent phases consume. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (shadcn deferred — no components needed in Phase 1; evaluate at Phase 2) |
| Preset | not applicable |
| Component library | Radix UI (install in Phase 2 when first interactive components are needed) |
| Icon library | Lucide React (standard pairing with Radix; install in Phase 2) |
| Font | Pretendard (Korean-optimized sans-serif with Latin coverage; loaded via CDN or @font-face) |

**Rationale:** CLAUDE.md recommends "Headless UI / Radix" for accessible UI primitives. Radix is chosen over Headless UI because shadcn/ui (likely to be adopted in Phase 2+) is built on Radix. Pretendard is selected because MiceSign is a Korean-language enterprise system — Pretendard provides excellent Hangul rendering with consistent Latin character metrics, available as a variable font.

**Phase 1 scope:** Only the TailwindCSS config file and font loading are delivered. No React components, no Radix install, no icon library yet.

## Spacing Scale

Declared values (multiples of 4, mapped to Tailwind spacing tokens):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1`, `gap-1` | Icon gaps, inline padding |
| sm | 8px | `p-2`, `gap-2` | Compact element spacing, badge padding |
| md | 16px | `p-4`, `gap-4` | Default element spacing, card padding |
| lg | 24px | `p-6`, `gap-6` | Section padding, form group spacing |
| xl | 32px | `p-8`, `gap-8` | Layout gaps between major sections |
| 2xl | 48px | `p-12`, `gap-12` | Major section breaks |
| 3xl | 64px | `p-16`, `gap-16` | Page-level vertical spacing |

Exceptions: Touch targets for interactive elements use minimum 44px height (Tailwind `h-11`) per WCAG 2.5.5 recommendation.

**Implementation:** Use Tailwind's default spacing scale (which is already 4px-based). No custom spacing tokens needed in `tailwind.config.js` — the default scale covers all values above.

## Typography

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|-----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-normal` |
| Label | 14px | 400 (regular) | 1.4 | `text-sm font-normal leading-snug tracking-wide uppercase` |
| Heading | 20px | 600 (semibold) | 1.3 | `text-xl font-semibold leading-tight` |
| Display | 28px | 600 (semibold) | 1.2 | `text-[28px] font-semibold leading-tight` |

**Label distinction strategy:** Label shares weight 400 with Body. Visual distinction is achieved through `tracking-wide` (0.025em letter-spacing) and `uppercase` transforms when contextually appropriate, or through color (`text-gray-600` for secondary labels vs `text-gray-900` for body). When labels appear alongside form inputs, the smaller context (field above input) provides sufficient distinction without a separate weight.

**Rationale:**
- Body at 14px: Enterprise approval systems are data-dense. 14px provides readability while fitting more content (document lists, approval lines, form fields) without scrolling. Korean characters render well at 14px with Pretendard.
- Two weights only (400 + 600): Regular for all running text (body, labels, captions) and semibold for all hierarchically prominent text (headings, display titles). This constraint reduces font file weight, simplifies design decisions, and produces a cleaner visual rhythm.
- Line heights tuned for Korean text which benefits from slightly more generous spacing than Latin-only text.

**Font stack in Tailwind config:**
```
fontFamily: {
  sans: ['Pretendard Variable', 'Pretendard', '-apple-system', 'BlinkMacSystemFont', 'system-ui', 'Helvetica Neue', 'Apple SD Gothic Neo', 'sans-serif']
}
```

## Color

| Role | Value | Tailwind | Usage |
|------|-------|----------|-------|
| Dominant (60%) | `#FFFFFF` / `#F9FAFB` | `bg-white` / `bg-gray-50` | Page background, content area surfaces |
| Secondary (30%) | `#F3F4F6` / `#E5E7EB` | `bg-gray-100` / `bg-gray-200` | Sidebar background, card backgrounds, table headers, dividers |
| Accent (10%) | `#2563EB` | `bg-blue-600` / `text-blue-600` | See reserved-for list below |
| Destructive | `#DC2626` | `bg-red-600` / `text-red-600` | Delete buttons, reject actions, error messages |

**Accent (`#2563EB` / blue-600) reserved for:**
1. Primary action buttons (submit, approve, save)
2. Active navigation item indicator
3. Links and interactive text
4. Focus ring on form inputs
5. Selected/active state highlights
6. Pending approval badge (as defined in FSD: `bg-blue-100 text-blue-700` for SUBMITTED status)

**Semantic status colors (from FSD Section 12.5 — locked):**

| Status | Background | Text | Tailwind |
|--------|------------|------|----------|
| DRAFT | gray | gray | `bg-gray-100 text-gray-700` |
| SUBMITTED | blue | blue | `bg-blue-100 text-blue-700` |
| APPROVED | green | green | `bg-green-100 text-green-700` |
| REJECTED | red | red | `bg-red-100 text-red-700` |
| WITHDRAWN | amber | amber | `bg-amber-100 text-amber-700` |

**Implementation:** Use Tailwind's default color palette. No custom colors in `tailwind.config.js` — Tailwind's `blue-600`, `red-600`, `green-600`, `amber-600`, and gray scale match the design contract exactly.

## Copywriting Contract

Phase 1 delivers no user-facing screens. The following are foundation-level copy decisions that apply to the entire application and will be consumed by subsequent phases.

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — Phase 1 has no interactive UI |
| Empty state heading | Not applicable — Phase 1 has no data views |
| Empty state body | Not applicable — Phase 1 has no data views |
| Error state | Not applicable — Phase 1 has no error-displaying UI |
| Destructive confirmation | Not applicable — Phase 1 has no destructive actions |

**Application-wide copy conventions (for future phases):**

| Convention | Decision |
|------------|----------|
| Language | Korean (한국어) for all UI text, labels, messages, and button copy |
| Honorific level | 합쇼체 (formal polite) for system messages, 해요체 (standard polite) for conversational UI text |
| Button pattern | `[동사] + [목적어]` format (e.g., "문서 상신", "결재 승인", not just "확인") |
| Error message pattern | `[무엇이 문제인지] + [어떻게 해결하는지]` (e.g., "파일 크기가 50MB를 초과합니다. 더 작은 파일을 선택해 주세요.") |

## Phase 1 Tailwind Config Contract

The following `tailwind.config.js` values must be set during Phase 1 scaffolding:

```javascript
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {
      fontFamily: {
        sans: [
          'Pretendard Variable',
          'Pretendard',
          '-apple-system',
          'BlinkMacSystemFont',
          'system-ui',
          'Helvetica Neue',
          'Apple SD Gothic Neo',
          'sans-serif',
        ],
      },
    },
  },
  plugins: [],
}
```

**Font loading:** Add Pretendard via CDN link in `index.html`:
```html
<link rel="stylesheet" as="style" crossorigin
  href="https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/variable/pretendardvariable.min.css" />
```
