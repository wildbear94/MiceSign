## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (p-1) | Icon gaps, inline padding, palette item icon-to-label gap |
| sm | 8px (p-2) | Compact element spacing, field card internal padding |
| md | 16px (p-4) | Default element spacing, panel padding, canvas padding |
| lg | 24px (p-6) | Section padding, modal body padding |
| xl | 32px (p-8) | Page-level vertical padding |

Exceptions:
- Palette panel width: 200px fixed (D-01)
- Property panel width: 300px fixed (D-01)
- Canvas: flex-1 (fluid, fills remaining space)
- Builder toolbar height: 56px (h-14) to match existing navbar height pattern
- Minimum viewport: 1024px (D-03)

## Interaction Contracts

### Three-Panel Layout (BLDR-01)
- Left palette: 200px fixed width, `overflow-y: auto`, scrollable when field types exceed viewport
- Center canvas: `flex: 1`, `min-width: 400px`, scrollable vertically
- Right property panel: 300px fixed width, `overflow-y: auto`
- Panels separated by 1px gray-200 borders (vertical dividers)
- No panel resizing (D-01)
- Below 1024px viewport: show full-page message "데스크톱 브라우저에서 사용해 주세요" centered with Monitor icon (D-03)

### Drag and Drop (BLDR-02, BLDR-03)
- Single `DragDropContext` wrapping all three panels
- Palette Droppable: `isDropDisabled={true}`, `droppableId="palette"`
- Canvas Droppable: `droppableId="canvas"`, accepts drops
- Palette-to-canvas: on drop, create new FieldDefinition with `nanoid()` ID, insert at destination index
- Canvas-to-canvas: on drop, reorder fields array
- Click-to-append: clicking palette item appends field at canvas bottom (D-04)
- Drop indicator: 2px blue-500 horizontal line between field cards at insertion point (D-07)
- Drag ghost from palette: semi-transparent copy (opacity-50) (D-07)
- Canvas drag handle: `GripVertical` icon on left side of each field card, `cursor: grab` on hover (D-08)

### Field Selection (BLDR-04, D-06)
- Click on field card: set `selectedFieldId`, show ring-2 ring-blue-600 border
- Click on canvas background: deselect (`selectedFieldId = null`), show template settings in property panel (D-14)
- Field card toolbar (appears on selected field): duplicate button (Copy icon) + delete button (Trash2 icon)
- Duplicate: new nanoid, label + " (복사본)", insert below selected field (D-09)
- Delete: remove from fields array, deselect, no confirmation dialog (single field, undo not in scope)

### Property Panel Tabs (BLDR-04, D-11)
- Three tabs using @headlessui/react Tab component:
  - **기본** (Basic): label, required toggle, placeholder, description
  - **검증** (Validation): min/max length (text/textarea), min/max value (number), pattern (text)
  - **고급** (Advanced): width (full/half radio), default value, field ID (read-only display)
- Tab content adapts to field type: hide irrelevant fields (e.g., no placeholder for table/staticText)
- When no field selected: show TemplateSettingsPanel (name, prefix, description, category, icon) (D-14)
- Select field type shows SelectOptionsEditor: inline value/label rows with add/remove + "옵션 세트 불러오기" button (D-12)
- Table field type shows TableColumnsEditor: column list with add/remove/reorder + per-column type/label/required editing (D-13)

### Live Preview (BLDR-05, D-02)
- Toggle button group in toolbar: "편집" / "프리뷰" (Edit / Preview)
- Edit mode: show canvas with field cards (default)
- Preview mode: replace canvas area with `DynamicForm` component rendered from current builder state's SchemaDefinition
- Preview renders exactly as end-users will see the form
- Property panel and palette remain visible but dimmed/disabled in preview mode (opacity-60, pointer-events-none)

### Template Management Page (BLDR-06, D-17)
- Table columns: Template name, Code (prefix), Field count, Version, Active status (badge), Last modified
- Row actions: Edit (navigate to builder page), Deactivate toggle
- "새 템플릿" button (top-right, blue-600 primary style) opens TemplateCreateModal (D-18)
- TemplateCreateModal fields: name (required), prefix/code (required), description (optional), category (optional)
- On create: navigate to `/admin/templates/:id/builder`
- Deactivate: ConfirmDialog with danger variant (D-20)
- Pagination: reuse existing Pagination component

### Builder Toolbar (D-22)
- Left: Back arrow button ("목록으로" - navigate to template list)
- Center: Template name display (read-only, editable in property panel settings)
- Right: JSON import/export dropdown + Preview toggle + Save button (primary blue)
- Save button: "저장" label, shows loading spinner during API call
- Unsaved changes indicator: small dot or asterisk next to template name when isDirty

### JSON Import/Export (D-25, D-26)
- Export: download current schema as `{template-code}-schema.json`
- Import flow: file upload input -> Zod validation -> show preview in modal -> user confirms -> apply to builder state
- Import validation errors: show list of specific errors in modal with red text
- Import success: replace current builder fields with imported schema fields

### Save Behavior (D-21)
- Immediate save on button click (no auto-save, no draft/publish separation)
- Success: brief green toast "템플릿이 저장되었습니다"
- Failure: red toast "저장에 실패했습니다. 다시 시도해 주세요" (D-24)
- After save: `isDirty` resets to false, schema version may increment (backend handles versioning)

## Field Width Rendering (D-15, D-16)

DynamicForm and DynamicReadOnly must be extended to support `config.width`:
- `"full"` (default): field takes full width (`w-full`)
- `"half"`: field takes 50% width (`w-1/2` inside a `flex flex-wrap gap-4` container)
- Two consecutive `half` fields render side-by-side in the same row
- A single `half` field followed by a `full` field: half field occupies left half, next row starts with full field
- Canvas in edit mode: does NOT render actual 2-column layout. Instead shows a purple "1/2" badge on half-width field cards. Real 2-column layout visible only in preview mode.

## Accessibility

- All drag-and-drop operations via @hello-pangea/dnd support keyboard (Space to lift, arrow keys to move, Space to drop)
- Tab component uses @headlessui/react Tab for automatic ARIA roles and keyboard navigation
- Field card selection: focus-visible ring on keyboard focus
- All form inputs in property panel have associated labels
- ConfirmDialog has focus trap and Escape to close (existing implementation)
- Desktop-only guard message: no action required, informational only
- Color contrast: all text meets WCAG AA (gray-600 on white = 5.74:1, blue-600 on white = 4.56:1)
