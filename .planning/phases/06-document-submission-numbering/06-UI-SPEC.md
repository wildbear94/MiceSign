## Interaction Contract

### Submit Flow

1. **Pre-condition:** Document must be saved (savedDocId is not null). Submit button is disabled for unsaved new documents.
2. **User clicks submit button** (SendHorizonal icon + "제출" label).
3. **Frontend validation runs** — same Zod schemas as save-time validation, plus any submit-specific checks (title required for all templates).
4. **If validation fails:** SubmitConfirmDialog opens with validation errors listed. Confirm button hidden. Only cancel/dismiss available.
5. **If validation passes:** SubmitConfirmDialog opens with confirmation message. Confirm button enabled.
6. **User confirms:** Submit mutation fires. Button shows loading spinner (Loader2 icon). Dialog is not dismissable during loading.
7. **On success:** Navigate to DocumentDetailPage for this document. Show document number and SUBMITTED status.
8. **On error:** Show error toast/inline message in dialog. Dialog remains open for retry or dismiss.

### Submit Button Placement

```
[<- 목록으로]                    [임시저장] [제출] [삭제]
```

- Submit button: `bg-blue-600 hover:bg-blue-700 text-white h-11 px-4 rounded-lg text-sm font-medium`
- Uses SendHorizonal icon (16x16, `h-4 w-4`) from lucide-react
- Positioned between save and delete buttons
- Disabled state: `opacity-50 cursor-not-allowed` when document is unsaved or mutation is pending

### Document Number Display — List Table

New column header: "문서번호" (i18n key: `columns.docNumber`)
- DRAFT rows: "-" in `text-gray-400 dark:text-gray-600`
- Other statuses: `font-mono text-sm text-gray-900 dark:text-gray-100`
- Column position: 2nd column (after title, before template)
- Column width: `w-40`

### Document Number Display — Detail Page

- Position: Below document title, above template badge
- DRAFT: Do not render document number element
- SUBMITTED and beyond: `font-mono text-base text-gray-500 dark:text-gray-400`
- Format example: "GEN-2026-0001"

### Read-Only Enforcement (Frontend)

When `document.status !== 'DRAFT'`:
- Editor components are NOT loaded (existing Phase 4 D-16 pattern)
- Submit button is NOT rendered
- Save button is NOT rendered
- Delete button is NOT rendered
- Attachment delete buttons are hidden (existing Phase 5 D-09)

## State Diagram — Submit Button

```
[Document unsaved] --> Submit button: disabled (opacity-50)
[Document saved, DRAFT] --> Submit button: enabled (blue-600)
[Submit clicked] --> Validation runs
  [Validation fails] --> Dialog with errors, no confirm
  [Validation passes] --> Dialog with confirm
    [User confirms] --> Loading state (Loader2 spinner)
      [Success] --> Navigate to detail page
      [Error] --> Error message in dialog, retry available
    [User cancels] --> Dialog closes, return to editor
[Document not DRAFT] --> Submit button: not rendered
```
