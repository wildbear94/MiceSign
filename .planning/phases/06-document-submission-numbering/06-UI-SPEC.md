## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, button gaps in action bar |
| md | 16px | Default element spacing, dialog internal padding |

| Value | Justification |

## Layout Specifications

### Editor Action Bar (Modified per D-01)

Current layout places save as primary (blue) and delete as secondary. Phase 6 changes button hierarchy:

```
+----------------------------------------------------------------------+
| [<- 돌아가기]                  [AutoSaveIndicator]                    |
|                                                                      |
|                                [삭제] [저장]  [제출]                  |
+----------------------------------------------------------------------+
```

| Button | Style | Visibility |
|--------|-------|------------|
| 돌아가기 (back) | `text-sm text-gray-500 hover:text-gray-700` with ArrowLeft icon | Always |
| 삭제 (delete) | `h-11 px-4 border border-gray-300 text-gray-700 rounded-lg hover:bg-gray-50` | Only when `savedDocId` exists |
| 저장 (save) | `h-11 px-4 border border-gray-300 text-gray-700 rounded-lg hover:bg-gray-50` | Always |
| 제출 (submit) | `h-11 px-4 bg-blue-600 hover:bg-blue-700 text-white text-sm font-medium rounded-lg` | Only when `savedDocId` exists AND status is DRAFT (D-04) |

Note: Save button changes from `bg-blue-600` (current) to `border border-gray-300 text-gray-700` (outline) per D-03. Submit takes the primary accent role.

## Interaction Contracts

### Submit Flow Sequence (D-01 through D-19)

| Step | Action | Result |
|------|--------|--------|
| 1 | User clicks "제출" button | Frontend validation runs (D-06, D-09) |
| 2a | Validation fails | Inline field errors displayed (D-08); flow stops |
| 2b | Validation passes | Check for upload-in-progress files (D-13) |
| 3a | Uploads in progress | Show "파일 업로드가 진행 중입니다" message; flow stops |
| 3b | Failed upload files exist | Show "업로드 실패한 파일이 있습니다" message (D-14); flow stops |
| 3c | No upload issues | Submit confirmation dialog opens (D-02) |
| 4 | User clicks "취소" in dialog | Dialog closes; no action |
| 5 | User clicks "제출" in dialog | Auto-save if unsaved changes (D-12); then submit API call |
| 6 | Submit API loading | Submit button shows spinner + disabled; form overlay at 50% opacity (D-05) |
| 7a | API returns success | Navigate to `/documents/{id}` with success state (D-17); show success banner (D-18) |
| 7b | API returns 409 Conflict | Show "이미 제출된 문서입니다" inline error (D-11) |
| 7c | API returns other error | Show "문서 제출에 실패했습니다" inline error (D-15) |

### Frontend Validation Rules (D-06)

| Template | Required Fields | Validation |
|----------|----------------|------------|
| GENERAL | title, bodyHtml | title is non-empty string; bodyHtml is non-empty and not just `<p></p>` |
| EXPENSE | title, expense items | title is non-empty; at least 1 expense item row with valid data |
| LEAVE | title, dates, reason | title is non-empty; startDate and endDate set; reason is non-empty |

Validation is performed by triggering the existing form's Zod schema validation (same as save validation) plus additional submit-specific checks.

### Submit Button State Machine

```
HIDDEN ──(savedDocId exists + DRAFT)──> ENABLED
                                            |
ENABLED ──(click)──> VALIDATING ──(fail)──> ENABLED + errors shown
                                    |
                              (pass)──> CONFIRMING (dialog open)
                                            |
                                    (cancel)──> ENABLED
                                    (confirm)──> SUBMITTING (loading)
                                                      |
                                              (success)──> REDIRECT
                                              (error)──> ENABLED + error shown
```

| State | Button Appearance |
|-------|-------------------|
| HIDDEN | Not rendered (non-DRAFT or unsaved new document) |
| ENABLED | `bg-blue-600 text-white` with text "제출" |
| SUBMITTING | `bg-blue-600 text-white opacity-50 cursor-not-allowed` with `Loader2` spinner |

### Success Message Auto-Dismiss (D-19)

| Property | Value |
|----------|-------|
| Display trigger | `location.state?.submitSuccess === true` on DocumentDetailPage mount |
| Duration | 5000ms via `setTimeout` |
| Manual dismiss | X button click sets message to null |
| Cleanup | `clearTimeout` on unmount; `window.history.replaceState` to clear location state after reading |

### Immutability Enforcement (DOC-04)

| Context | Behavior |
|---------|----------|
| DocumentDetailPage for non-DRAFT | Read-only view (already implemented in Phase 4 D-16) |
| DocumentEditorPage | Only accessible for DRAFT documents; non-DRAFT redirects to detail page |
| File attachments | Delete button hidden for non-DRAFT documents (Phase 5 D-09) |
| Submit button | Hidden for non-DRAFT documents (D-04) |
