<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> API | Untrusted approval line data crosses to backend (approverId, lineType, stepOrder) |
| approver -> approval action | User claims to be the current approver to approve/reject |
| drafter -> withdraw | User claims to be the drafter and that no approver has acted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Spoofing | ApprovalService.approve/reject | mitigate | validateApprovalAction() checks userId matches current step approver [VERIFIED existing] |
| T-07-02 | Tampering | DocumentService.submitDocument | mitigate | Approval line validation: APPROVE >= 1, drafter not in line [activated in this plan] |
| T-07-03 | Tampering | DocumentService.withdrawDocument | mitigate | Backend checks status=SUBMITTED, userId=drafter, currentStep approvers all PENDING [VERIFIED existing] |
| T-07-04 | Elevation | ApprovalService.approve | mitigate | stepOrder == document.currentStep check [VERIFIED existing] |
| T-07-05 | Information Disclosure | DocumentService.getDocument | mitigate | RBAC check: drafter + approval line users + admin only [VERIFIED existing] |
| T-07-06 | Repudiation | ApprovalService | mitigate | AuditLogService records all approval actions [VERIFIED existing] |
</threat_model>
