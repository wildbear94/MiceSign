## Layout Specifications

### Approval Line Editor (in DocumentEditorPage, below attachments)

```
+---------------------------------------------------------------+
| 결재선 설정                                    [접기/펼치기]    |
+------------------------------+--------------------------------+
| 조직도 (left panel)          | 결재선 (right panel)           |
| +--------------------------+ | +----------------------------+ |
| | [>] 경영지원본부         | | | 1. 김부장 (경영지원본부)   | |
| |    [>] 인사팀            | | |    [승인 v] [x]  [≡ drag] | |
| |       홍길동 사원 [+]    | | | 2. 이과장 (개발팀)         | |
| |       김영희 대리 [+]    | | |    [합의 v] [x]  [≡ drag] | |
| |    [>] 총무팀            | | | ─── 참조 ─────────────── | |
| |       ...                | | | 박대리 (총무팀) [x]        | |
| +--------------------------+ | +----------------------------+ |
+------------------------------+--------------------------------+
```

- Left panel width: 50% on desktop, full width stacked on mobile (<768px)
- Right panel width: 50% on desktop, full width stacked on mobile
- Min-height: 200px
- Border: `border border-gray-200 dark:border-gray-700 rounded-xl`
- Section heading: `text-sm font-semibold text-gray-700 dark:text-gray-300`
- Panel divider: `border-r border-gray-200 dark:border-gray-700` (desktop only)

### Approval Line Item (draggable row)

```
+------------------------------------------------------+
| [≡] 1. 홍길동 (경영지원본부 / 부장)  [승인 v]  [x]  |
+------------------------------------------------------+
```

- Height: 44px minimum (touch target)
- Drag handle: `GripVertical` icon from lucide-react, `text-gray-400 cursor-grab`
- Step number: `text-sm font-medium text-gray-500`
- Name: `text-sm font-medium text-gray-900 dark:text-gray-50`
- Dept/position: `text-sm text-gray-500 dark:text-gray-400`
- Type dropdown: `text-xs` select with border, width 72px
- Remove button: `X` icon, `text-gray-400 hover:text-red-500`
- REFERENCE items: no step number, rendered below separator line
- Separator line between APPROVE/AGREE and REFERENCE: `border-t border-dashed border-gray-300 dark:border-gray-600` with centered label "참조"


  - Future pending: `Circle` in `text-gray-300 dark:text-gray-600`

  - Skipped: `MinusCircle` in `text-gray-400`

### Approval Action Bar (top of DocumentDetailPage for submitted docs)

```
+---------------------------------------------------+
| [결재하기]  [반려하기]                 [문서 회수]  |
+---------------------------------------------------+
```

- Position: Between back-link and meta info section
- Approve button: `bg-blue-600 hover:bg-blue-700 text-white h-11 px-5 rounded-lg text-sm font-medium`
- Reject button: `bg-white border border-red-300 text-red-600 hover:bg-red-50 h-11 px-5 rounded-lg text-sm font-medium` (outlined, not filled)
- Withdraw button: `bg-white border border-gray-300 text-gray-700 hover:bg-gray-50 h-11 px-5 rounded-lg text-sm font-medium` (right-aligned)
- Resubmit button: `bg-blue-600 hover:bg-blue-700 text-white h-11 px-5 rounded-lg text-sm font-medium` (on REJECTED/WITHDRAWN docs, for drafter only)
- Visibility rules:
  - Approve/Reject: only for current-turn approver, only on SUBMITTED docs
  - Withdraw: only for drafter, only on SUBMITTED docs, only when next approver has not acted
  - Resubmit: only for drafter, only on REJECTED/WITHDRAWN docs

## Interaction Contracts

### Drag-and-Drop (Approval Line Reordering)

- Library: `@hello-pangea/dnd` 18.0.1 (already installed)
- Only APPROVE/AGREE items are draggable; REFERENCE items are not draggable
- Drag placeholder: `bg-blue-50 dark:bg-blue-900/20 border-2 border-dashed border-blue-300 rounded-lg`
- On drop: recompute visual step numbers (1, 2, 3...) for APPROVE/AGREE items
- Keyboard support: space to grab, arrow keys to move, space to drop (built-in with @hello-pangea/dnd)

### Org Tree User Selection

- Expand department node to reveal users under that department
- Each user row shows: name, position name, and an add button [+]
- Add button click opens a small inline dropdown (APPROVE/AGREE/REFERENCE) next to the button
- After type selection, user is added to right panel
- Already-added users: add button disabled with `opacity-50 cursor-not-allowed`
- Self (current logged-in user): add button disabled, tooltip "본인은 결재선에 추가할 수 없습니다"

### Tooltip Implementation

- Use `title` attribute for simple tooltips (D-10 type explanations)
- APPROVE tooltip: "결재권자 — 문서를 승인 또는 반려합니다"
- AGREE tooltip: "합의자 — 협조 부서의 동의를 구합니다"
- REFERENCE tooltip: "참조자 — 열람만 가능합니다"

### State Transitions (UI Feedback)

| Action | Loading State | Success | Error |
|--------|--------------|---------|-------|
| Approve | Button shows Loader2 spinner, disabled | Dialog closes, page refreshes, toast/redirect | Error message in dialog |
| Reject | Button shows Loader2 spinner, disabled | Dialog closes, page refreshes | Error message in dialog |
| Withdraw | Button shows Loader2 spinner, disabled | Redirect to my documents list | Error message in dialog |
| Resubmit | Button shows Loader2 spinner, disabled | Redirect to new draft editor page | Error message |
