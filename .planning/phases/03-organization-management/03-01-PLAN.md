**DepartmentService** (`service/DepartmentService.java`):
- `@Service @Transactional(readOnly = true)` class.
- **getDepartmentTree(boolean includeInactive)**: Fetch all departments (or active only), compute member counts via `departmentRepository.countActiveUsersByDepartment()` into `Map<Long, Integer>`, build recursive tree from flat list. Root nodes have `parentId == null`. Sort children by `sortOrder`. Return `List<DepartmentTreeResponse>`.
- **getDepartmentMembers(Long departmentId)**: Fetch `userRepository.findByDepartmentIdAndStatus(id, ACTIVE)`, map to `List<DepartmentMemberResponse>` including position name from the `@ManyToOne` relationship.
- **createDepartment(CreateDepartmentRequest)** `@Transactional`: Validate name uniqueness (`existsByName` -> throw `ORG_DUPLICATE_NAME`). If parentId provided: validate parent exists and is active, validate depth (calculate parent depth, must be < 2 so new node is at max level 2, 0-indexed = 3 levels per D-09). Create entity, save, return tree node response.
- **updateDepartment(Long id, UpdateDepartmentRequest)** `@Transactional`: Find by ID or throw `ORG_NOT_FOUND`. Validate name unique excluding self (`existsByNameAndIdNot`). If parentId changed: validate no circular reference by walking ancestors from new parent up to root — if `id` is encountered, throw `ORG_CIRCULAR_REF`. Validate depth of entire subtree under new parent won't exceed 3. Save, return updated response.
- **deactivateDepartment(Long id)** `@Transactional`: Check `existsByParentIdAndIsActiveTrue(id)` — if true, throw `ORG_HAS_ACTIVE_CHILDREN`. Set `isActive = false`, save.
- **getUserCountByDepartment(Long departmentId)**: Return count of active users in department via `userRepository.countByDepartmentIdAndStatus(departmentId, ACTIVE)`.
- Private helper `calculateDepth(Long departmentId)`: Walk up parent chain counting steps. Return 0 for root.
- Private helper `validateNoCircularReference(Long departmentId, Long newParentId)`: Walk up from newParentId. If departmentId found, throw `ORG_CIRCULAR_REF`.
- Private helper `calculateMaxSubtreeDepth(Long departmentId)`: BFS/DFS to find deepest descendant depth. Used when moving to ensure new position + subtree depth <= 3.

**PositionService** (`service/PositionService.java`):
- **getAllPositions()**: Fetch all ordered by sortOrder, compute user counts per position, return `List<PositionResponse>`.
- **createPosition(CreatePositionRequest)** `@Transactional`: Validate name uniqueness (`existsByName` -> `ORG_DUPLICATE_NAME`). Auto-assign sortOrder as `max(sortOrder) + 1` from existing positions. Save, return response.
