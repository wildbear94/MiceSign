## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding (`px-1`) |
| sm | 8px | Compact element spacing (`px-2`, `py-2`), tab padding |
| md | 16px | Default element spacing (`space-y-4`, `px-4`, `gap-4`), filter row gaps |
| lg | 24px | Section margin between tabs/filters/table (`mb-6`) |
| xl | 32px | Not used in this phase |

Exceptions: Touch target height is 44px (`h-11`) for search input and filter inputs, matching existing `AuditLogFilters` pattern. Tab buttons use 40px (`h-10`) height for comfortable click targets.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `gray-900` (dark) | Page background, input backgrounds, table body |
| Secondary (30%) | `gray-50` / `gray-800` (dark) | Table header, filter area background (optional), hover rows |
| Accent (10%) | `blue-600` / `blue-500` | Active tab indicator, search button, focus rings, "New Document" CTA |
| Destructive | `red-500` / `red-600` | Not used in this phase (no destructive actions) |

Accent reserved for:
- Search button background (`bg-blue-600 hover:bg-blue-700 text-white`)
- Active tab text and bottom border (`text-blue-600 border-b-2 border-blue-600`)
- Input focus ring (`focus:ring-2 focus:ring-blue-500`)
- "New Document" button (`bg-blue-600`)
- Search keyword highlight background (`bg-yellow-200 dark:bg-yellow-800`) -- yellow, not accent blue, to distinguish from interactive elements

Highlight color:
- Search match: `bg-yellow-200 dark:bg-yellow-800 rounded px-0.5` on `<mark>` element

Border colors:
- Input borders: `border-gray-300 dark:border-gray-600`
- Table borders: `border-gray-200 dark:border-gray-700`
- Tab underline (inactive): `border-gray-200 dark:border-gray-700` (bottom border on tab container)

Text colors:
- Primary text: `text-gray-900 dark:text-gray-50`
- Table header text: `text-gray-600 dark:text-gray-400`
- Tab inactive text: `text-gray-500 dark:text-gray-400`
- Placeholder text: `text-gray-400 dark:text-gray-500` (via `placeholder:` modifier)
- Empty state text: `text-gray-500 dark:text-gray-400`
- Muted info text: `text-gray-400 dark:text-gray-500`

Source: All values from existing `AuditLogFilters.tsx`, `DocumentListPage.tsx`, `DocumentListTable.tsx`. Highlight color from RESEARCH.md recommendation.

## Interaction Contracts

### Search Behavior

| Interaction | Behavior |
|-------------|----------|
| Type in search input + Enter | Trigger search with current keyword + filters. Reset page to 0. |
| Click search button | Same as Enter. |
| Clear search input + Enter | Remove keyword filter, show all results in current tab/filters. |
| Type in search input (no Enter) | No API call. Local state only. |

### Filter Behavior

| Interaction | Behavior |
|-------------|----------|
| Change status dropdown | Update local filter state. Does NOT trigger search. |
| Change template dropdown | Update local filter state. Does NOT trigger search. |
| Change date inputs | Update local filter state. Does NOT trigger search. |
| Click "검색" button | Apply all local filter state + keyword to URL params, trigger API call. Reset page to 0. |
| Click "초기화" button | Clear all local filter state (keyword, status, template, dates). Do NOT trigger search. User must click "검색" to apply the reset. |

### Tab Behavior

| Interaction | Behavior |
|-------------|----------|
| Click tab | Update `tab` in URL params. Clear keyword and all filters. Reset page to 0. Trigger new API call. |
| ADMIN/SUPER_ADMIN loads page | 3 tabs visible. |
| USER loads page | 2 tabs visible (전체 hidden). |
| USER navigates to `?tab=ALL` directly | Backend returns 403. Frontend redirects to `?tab=MY`. |

### URL State Sync

| URL Parameter | Default | Cleared When |
|---------------|---------|--------------|
| `tab` | `MY` | Never (always present or defaults) |
| `keyword` | (empty) | Tab change, reset button |
| `status` | (empty) | Tab change, reset button |
| `templateCode` | (empty) | Tab change, reset button |
| `startDate` | (empty) | Tab change, reset button |
| `endDate` | (empty) | Tab change, reset button |
| `page` | `0` | Any filter/search change, tab change |

### Keyword Highlighting

| Rule | Detail |
|------|--------|
| Highlight scope | title, docNumber, drafterName columns only |
| Match style | `<mark className="bg-yellow-200 dark:bg-yellow-800 rounded px-0.5">` |
| Case sensitivity | Case-insensitive match |
| No keyword | Render plain text (no `<mark>` wrapper) |
| Special characters | Escape regex special chars in keyword before splitting |

### Loading States

| State | Display |
|-------|---------|
| Initial load | Skeleton rows (`h-12 bg-gray-200 dark:bg-gray-700 rounded animate-pulse`) x 5 |
| Tab switch | Same skeleton rows |
| Filter search | Same skeleton rows (placeholderData from TanStack Query prevents flicker on pagination) |
| Pagination | Smooth transition via `placeholderData: (prev) => prev` |

### Accessibility

| Element | Attribute | Value |
|---------|-----------|-------|
| Search input | `placeholder` | `"검색어를 입력하세요 (제목, 문서번호, 기안자)"` |
| Search input | `aria-label` | `"문서 검색"` |
| Status select | `aria-label` | `"상태 필터"` |
| Template select | `aria-label` | `"양식 필터"` |
| Start date input | `aria-label` | `"시작일"` |
| End date input | `aria-label` | `"종료일"` |
| Reset button | text content | `"초기화"` (visible label) |
| Search button | text content | `"검색"` (visible label) |
| Tab buttons | `role` | `"tab"` (via `role="tablist"` on container) |
| Active tab | `aria-selected` | `"true"` |
| Search icon in input | `aria-hidden` | `"true"` |
