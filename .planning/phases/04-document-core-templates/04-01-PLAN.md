**DocumentFormValidator** (`@Component`), per D-13:
- `public void validate(String templateCode, String bodyHtml, String formDataJson)`:
  - `GENERAL`: bodyHtml must not be blank. formData ignored (null OK).
  - `EXPENSE`: Parse formDataJson as JSON. Validate structure: must have `items` (array, min 1 element), each item has `name` (non-blank string), `quantity` (positive int), `unitPrice` (non-negative long), `amount` (non-negative long). Must have `totalAmount` (non-negative long). Optionally may have `paymentMethod` and `accountInfo` (per FSD). Throw `BusinessException("DOC_INVALID_FORM_DATA", "...")` with Korean message on failure.
  - `LEAVE`: Parse formDataJson as JSON. Validate: `leaveTypeId` (positive long), `startDate` (non-blank string, ISO date format), `endDate` (non-blank string for non-half-day, nullable for half-day), `days` (positive number), `reason` (non-blank string). Optionally `startTime`, `endTime` for half-day. Throw on invalid.
  - Unknown templateCode: throw `BusinessException("TPL_UNKNOWN", "알 수 없는 양식 코드입니다: " + templateCode)`.
- Use `com.fasterxml.jackson.databind.ObjectMapper` (inject via constructor) to parse JSON. Use `JsonNode` for validation rather than deserializing to typed objects.

**DocumentService** (`@Service @Transactional`):
- `createDocument(Long userId, CreateDocumentRequest req) -> DocumentResponse`:
  1. Validate templateCode exists in approval_template table (else throw `BusinessException("TPL_NOT_FOUND", "양식을 찾을 수 없습니다.")`)
  2. Call `formValidator.validate(req.templateCode(), req.bodyHtml(), req.formData())`
  3. Create Document entity: set templateCode, title, drafter (load User by userId), status=DRAFT
  4. Save Document
  5. Create DocumentContent: set document, bodyHtml, formData
  6. Save DocumentContent
  7. Return mapped DocumentResponse via DocumentMapper

- `updateDocument(Long userId, Long documentId, UpdateDocumentRequest req) -> DocumentResponse`:
  1. Load document, verify `document.getDrafter().getId().equals(userId)` else throw `BusinessException("DOC_NOT_OWNER", "본인의 문서만 수정할 수 있습니다.")`
  2. Verify `document.getStatus() == DRAFT` else throw `BusinessException("DOC_NOT_DRAFT", "임시저장 상태의 문서만 수정할 수 있습니다.")`
  3. Call `formValidator.validate(document.getTemplateCode(), req.bodyHtml(), req.formData())`
  4. Update title, content (bodyHtml, formData)
  5. Return mapped response

- `deleteDocument(Long userId, Long documentId)`:
  1. Load document, verify owner, verify DRAFT status (same checks as update)
  2. Delete DocumentContent first, then Document (per D-18: physical hard delete)

- `getMyDocuments(Long userId, Pageable pageable) -> Page<DocumentResponse>`:
  1. `documentRepository.findByDrafterId(userId, pageable)`
  2. Map each to DocumentResponse (include templateName from a cached Map or join)

- `getDocumentDetail(Long userId, Long documentId) -> DocumentDetailResponse`:
  1. Load document with content
  2. For now, any authenticated user who is the drafter can view. (Phase 7 adds approver access.)
  3. Return full detail including bodyHtml, formData, drafter info
