## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding |
| sm | 8px | Compact element spacing, table cell padding |
| md | 16px | Default element spacing, form field gaps, input horizontal padding |
| lg | 24px | Section padding, card padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page top/bottom margin |
| 3xl | 64px | Not used this phase |

Exceptions:

| Value | Justification |
|-------|---------------|
| 44px (`h-11`) | Touch target minimum per WCAG 2.5.8. Used for all interactive input elements (title input, select dropdown, date inputs) and buttons (ConfirmDialog, Pagination). |

Source: Existing codebase uses Tailwind `p-6` (24px), `gap-2` (8px), `mb-6` (24px), `mb-4` (16px). Contract formalizes this. All horizontal input padding standardized to `px-4` (16px) to stay within declared scale.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Title Input | 16px | 600 (semibold) | 1.4 | `text-base font-semibold` |
| Subheading | 18px | 600 (semibold) | 1.3 | `text-lg font-semibold` |
| Heading | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold` |

Declared sizes: 14px, 16px, 18px, 24px (4 sizes — maximum allowed).

Declared weights: 400 (regular) for body text, 600 (semibold) for title input, subheadings, and headings. Two weights only.

Source: Existing pattern — `UserListPage.tsx` uses `text-2xl font-semibold` for page title, `text-sm` for body/labels. `ConfirmDialog.tsx` uses `text-lg font-semibold` for dialog titles. 16px added for title input and expense total row where `text-base` is used. Labels share 14px Body size with `font-semibold` — the weight distinguishes them from body text.

## Interaction Contracts

### Auto-Save (D-17)

| Property | Value |
|----------|-------|
| Trigger | Any form field change (title, body, form_data) |
| Debounce | 30,000ms (30 seconds) of inactivity |
| Indicator position | Top right of editor area, inline with action buttons |
| Indicator states | "저장됨" (gray-500, with Check icon), "저장 중..." (blue-600, with Loader2 spinning), "저장 실패" (red-600, with AlertCircle icon) |
| Manual save | Always visible "임시저장" button alongside auto-save indicator |
| New document | First auto-save creates the document (POST), subsequent saves update (PUT) |

### Template Selection Modal (D-14)

| Property | Value |
|----------|-------|
| Trigger | Click "새 문서" button |
| Close | Click backdrop, press Escape, or select a template |
| Focus trap | Yes (follow ConfirmDialog pattern) |
| Selection action | Navigate to `/documents/new/{templateCode}` |

### Expense Table (D-04, D-05, D-06)

| Property | Value |
|----------|-------|
| Add row | Click "항목 추가" appends empty row at bottom |
| Delete row | Click X button on row. Minimum 1 row required (disable delete when only 1 row). |
| Amount calculation | `amount = quantity * unitPrice`, recalculated on every cell blur/change |
| Total calculation | Sum of all row amounts, displayed in footer |
| Currency display | `Intl.NumberFormat('ko-KR', { style: 'currency', currency: 'KRW', maximumFractionDigits: 0 })` |
| Input behavior | Number inputs accept digits only, display formatted on blur |

### Leave Form (D-07, D-08, D-09)

| Property | Value |
|----------|-------|
| Leave type | Select dropdown populated from `/api/v1/leave-types` API |
| Half-day detection | When selected leave type has `isHalfDay: true`, show time picker fields instead of end date |
| Full-day calculation | `differenceInCalendarDays(endDate, startDate) + 1` (inclusive) |
| Half-day calculation | Fixed 0.5 days |
| Days display | Auto-updated on date/type change, displayed as accent color text |

### Document List (D-10)

| Property | Value |
|----------|-------|
| Default sort | Created date descending (newest first) |
| Row click | Navigate to `/documents/:id` |
| Columns | 제목, 양식, 상태, 작성일 |
| Pagination | Reuse existing `Pagination` component, 20 items per page |
| Loading state | Skeleton rows (5 animated pulse bars) per UserListPage pattern |

### Document Detail (D-11, D-16)

| Property | Value |
|----------|-------|
| DRAFT status | Render edit form (same as editor) |
| Non-DRAFT status | Render read-only template component |
| Approval line section | Dashed border placeholder with text |
| Attachment section | Dashed border placeholder with text |

### Inline Messages (D-20)

| Property | Value |
|----------|-------|
| Position | Below form actions, above form content |
| Success | `text-green-600` with Check icon |
| Error | `text-red-600` with AlertCircle icon |
| Duration | Success messages fade after 3 seconds, errors persist until next action |
