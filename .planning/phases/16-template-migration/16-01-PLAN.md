4개 파일을 수정하여 동적 템플릿 문서의 백엔드 파이프라인을 완성한다.

**(A) DocumentContent.java — JPA 필드 매핑 추가:**

formData 필드 아래에 두 필드를 추가:
```java
@Column(name = "schema_version")
private Integer schemaVersion;

@Column(name = "schema_definition_snapshot", columnDefinition = "LONGTEXT")
private String schemaDefinitionSnapshot;
```
getter/setter도 추가:
- `public Integer getSchemaVersion()` / `public void setSchemaVersion(Integer schemaVersion)`
- `public String getSchemaDefinitionSnapshot()` / `public void setSchemaDefinitionSnapshot(String schemaDefinitionSnapshot)`

(DB 컬럼은 V8 마이그레이션으로 이미 존재하므로 DDL 변경 불필요)

**(B) DocumentFormValidator.java — 동적 스키마 폴백:**

validate 메서드의 시그니처를 변경하여 template를 받도록 한다:
```java
public void validate(String templateCode, String bodyHtml, String formDataJson, String schemaDefinition)
```

switch의 default 케이스를 수정:
- schemaDefinition이 null이 아니면, JSON 파싱 후 required 필드들이 formDataJson에 존재하는지 기본 검증 수행. 구체적으로:
  1. schemaDefinition을 Jackson ObjectMapper로 파싱하여 fields 배열 추출
  2. formDataJson을 파싱하여 JsonNode 생성
  3. 각 field에서 required=true인 필드의 id가 formData에 존재하고 빈 문자열이 아닌지 확인
  4. table 타입 필드는 배열이고 minRows 이상인지 확인
- schemaDefinition이 null이면 기존대로 TPL_UNKNOWN 예외를 던짐

기존 GENERAL/EXPENSE/LEAVE 케이스는 그대로 유지 (per D-01, 하드코딩 밸리데이터 보존).

**(C) DocumentService.java — createDocument에 스냅샷 저장:**

createDocument 메서드에서:
1. template 객체를 이미 조회하고 있으므로, `template.getSchemaDefinition()`이 null이 아니면:
   - `content.setSchemaVersion(template.getSchemaVersion())`
   - `content.setSchemaDefinitionSnapshot(template.getSchemaDefinition())`
2. formValidator.validate 호출 시 4번째 인자로 `template.getSchemaDefinition()` 전달

updateDocument 메서드에서도 formValidator.validate 호출 부분을 동일하게 수정 (template 조회 후 schemaDefinition 전달).

**(D) DocumentDetailResponse.java — schemaDefinitionSnapshot 필드 추가:**

record 파라미터에 추가:
```java
public record DocumentDetailResponse(

    String title, String status, DrafterInfo drafter,
    String bodyHtml, String formData,
    String schemaDefinitionSnapshot,
    LocalDateTime submittedAt, LocalDateTime completedAt,
    LocalDateTime createdAt, LocalDateTime updatedAt
) { ... }
```

**(E) DocumentMapper.java — 매핑 추가:**

toDetailResponse 메서드에 매핑 추가:
```java
@Mapping(target = "schemaDefinitionSnapshot", source = "content.schemaDefinitionSnapshot")
```
