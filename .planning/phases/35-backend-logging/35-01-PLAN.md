추가할 정확한 yaml 블록 (들여쓰기 2-space, 기존 파일 컨벤션 일치):

```yaml
# Phase 35 — 백엔드 로그 설정 (logback 일별 롤링 + 30일 보관 + prod/dev 프로필 분리)
# D-A1: logback-spring.xml 미생성 — Spring Boot 3.x 표준 logging.* properties 만 사용
# D-B1: ${LOG_DIR:-logs} 로 dev=프로젝트 루트 상대경로, prod=systemd EnvironmentFile 의 LOG_DIR 주입
# D-D1: root=INFO 가 base 기본 (운영 기본값과 동일) — dev 프로필에서 DEBUG 로 override
logging:
  file:
    name: ${LOG_DIR:-logs}/micesign.log
  charset:
    file: UTF-8
  level:
    root: INFO
  logback:
    rollingpolicy:
      file-name-pattern: ${LOG_DIR:-logs}/micesign-%d{yyyy-MM-dd}.%i.log.gz
      max-history: 30
      max-file-size: 100MB
      total-size-cap: 1GB
      clean-history-on-start: true
```

배치 위치: 기존 `spring:` 블록 (L1-36) 종료 후, `jwt:` 블록 (L38-) 시작 전 — 즉 L37 빈 줄 자리에 위 블록을 삽입하고 jwt 블록 앞에 빈 줄 1개 유지.

WHY 이 키들:
- `logging.file.name` (D-B1): 전체 경로 명시. `logging.file.path` 는 디렉터리만 — name 사용해 micesign.log 파일명까지 고정.
- `${LOG_DIR:-logs}` 패턴: dev/local 에서는 `logs` (프로젝트 루트 상대), prod 에서는 systemd EnvironmentFile 의 `LOG_DIR=/var/log/micesign` 같이 주입.
- `logging.charset.file: UTF-8` (D-E2): 한글 결재 문서 제목/사용자 이름이 로그에 깨지지 않게.
- `logging.level.root: INFO` (D-D1): 운영 기본값. 명시함으로써 dev 가 DEBUG 로 override 하는 의도가 표면화됨.
- `file-name-pattern` 의 `%d{yyyy-MM-dd}.%i.log.gz` (D-C1): `%d` = 일별 롤링 트리거, `%i` = 동일 일자 내 100MB cap 도달 시 인덱스 안전망, `.gz` = Spring Boot 자동 gzip 압축 (Q4=gzip).
- `max-history: 30` (D-C2): 사용자 명시 "최대 30일 보관".
- `max-file-size: 100MB` (D-C3): 폭주 시 단일 파일 상한 (안전망 보조).
- `total-size-cap: 1GB` (D-C4): 전체 로그 디렉터리 디스크 사용 상한 (Q5=1GB). 50인 환경 + 30일 보관에 안전 마진 충분.
- `clean-history-on-start: true` (D-C5): 앱 재시작 시 max-history 초과 파일 즉시 삭제 — 서버 재기동 후 정합성 보장.
