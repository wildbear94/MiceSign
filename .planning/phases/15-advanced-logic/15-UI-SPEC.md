## Layout Contract

### PropertyConditionsTab Layout

```
+------------------------------------------+
| 조건부 규칙                                |  <- Label (text-xs semibold)
+------------------------------------------+
| 일치 조건: [모두 충족 v] / [하나 이상 v]     |  <- matchType toggle (select)
+------------------------------------------+
| +--------------------------------------+ |
| | [소스 필드 v] [연산자 v] [값 입력]  [X] | |  <- ConditionRuleCard
| +--------------------------------------+ |
| +--------------------------------------+ |
| | [소스 필드 v] [연산자 v] [값 입력]  [X] | |  <- ConditionRuleCard
| +--------------------------------------+ |
+------------------------------------------+
| [+ 조건 추가]                              |  <- accent button (text-sm)
+------------------------------------------+
| 실행 액션: [표시 v] / [숨기기 v] / [필수 v]  |  <- action select
+------------------------------------------+
```

- Container: `space-y-4 p-4`
- Each ConditionRuleCard: `flex items-center gap-2 p-3 bg-gray-50 dark:bg-gray-800 rounded-lg border border-gray-200 dark:border-gray-700`
- Source field dropdown: `flex-1 min-w-0` (shows field labels from current template)
- Operator dropdown: `w-32` fixed width
- Value input: `flex-1 min-w-0` (hidden when operator is `is_empty` or `is_not_empty`)
- Delete button: `p-1 text-gray-400 hover:text-red-500` with `X` icon (16px)

### CalculationConfig in PropertyAdvancedTab

```
+------------------------------------------+
| -- existing width/defaultValue fields --  |
+------------------------------------------+
| 자동 계산                                  |  <- section label (text-xs semibold)
+------------------------------------------+
| 연산: [SUM v]                             |  <- operation select
| 소스 필드:                                 |  <- label
| [v] expense_table.amount                  |  <- multi-select checkboxes
| [v] field_extra_cost                      |
| [ ] field_discount                        |
+------------------------------------------+
```

- Only visible when selected field type is `number`
- Separator: `border-t border-gray-200 dark:border-gray-700 pt-4 mt-4`
- Operation select: standard `<select>` with options: SUM, ADD, MULTIPLY, COUNT
- Source fields: grouped list with checkboxes -- "숫자 필드" group header + "테이블 칼럼" group header
- When calculation is configured, field becomes readonly in form -- show `bg-blue-50 text-blue-700 text-xs px-2 py-0.5 rounded` badge "자동 계산" next to field label in canvas

### DynamicSectionField Layout (User Form)

```
+----------------------------------------------------------+
| [v] 경비 상세 정보                                         |  <- collapsible section header
+----------------------------------------------------------+
|   [field 1]                                               |
|   [field 2]                                               |
|   [field 3]                                               |
+----------------------------------------------------------+
```

- Header button: `w-full flex items-center justify-between py-3 px-4 bg-gray-50 dark:bg-gray-800 rounded-lg border border-gray-200 dark:border-gray-700 hover:bg-gray-100 dark:hover:bg-gray-700 transition-colors`
- Label: `text-base font-semibold text-gray-900 dark:text-gray-100`
- Chevron icon: `h-5 w-5 text-gray-400 transition-transform duration-200` -- rotates `-rotate-90` when collapsed
- Children container: `mt-2 space-y-4 pl-4` with `overflow-hidden` transition
- Collapse animation: CSS `transition-all duration-200 ease-in-out` with max-height toggle pattern

### Section in Builder Canvas

```
+----------------------------------------------------------+
| [drag] [Rows3] 경비 상세 정보              [section badge] |  <- section field card
+----------------------------------------------------------+
```

- Section card in canvas: same FieldCard styling but with `border-l-4 border-blue-400` left accent to distinguish from regular fields
- Badge: `text-xs bg-blue-50 text-blue-600 px-1.5 py-0.5 rounded`

### Circular Dependency Error Toast

```
+----------------------------------------------------------+
| [AlertTriangle] 순환 의존성이 감지되었습니다                  |
| 필드A -> 필드B -> 필드A                                    |
+----------------------------------------------------------+
```

- Position: top-center, fixed
- Style: `bg-red-50 dark:bg-red-900/30 border border-red-200 dark:border-red-800 rounded-lg p-4 shadow-lg`
- Icon: `AlertTriangle` from Lucide, `text-red-500 h-5 w-5`
- Title: `text-sm font-semibold text-red-700 dark:text-red-400`
- Cycle path: `text-xs text-red-600 dark:text-red-300 font-mono mt-1`
- Auto-dismiss: 8 seconds (longer than default due to technical content)

## Interaction Contract

### Conditions Tab Interactions

| Interaction | Behavior |
|-------------|----------|
| Select field in canvas | PropertyPanel shows; if field has conditional rules, Conditions tab shows them |
| Click "조건 추가" | Append new empty condition row (source empty, operator "equals", value empty) |
| Change source field dropdown | Update condition in builder state via reducer |
| Change operator to is_empty/is_not_empty | Hide value input (value not needed) |
| Click delete (X) on condition | Remove condition row immediately (no confirmation) |
| Change matchType | Toggle between all/any for this field's rules |
| Change action | Toggle between show/hide/require |
| No conditions present | Show empty state text with guidance |

### Calculation Config Interactions

| Interaction | Behavior |
|-------------|----------|
| Select number field in canvas | PropertyAdvancedTab shows calculation section |
| Select non-number field | Calculation section is hidden |
| Select operation | Set calculationType on field config |
| Toggle source field checkbox | Add/remove sourceField from calculation rule |
| Calculation configured | Canvas shows "자동 계산" badge on field card |

### Dynamic Form Runtime Interactions

| Interaction | Behavior |
|-------------|----------|
| Source field value changes | useWatch triggers re-evaluation; target fields show/hide/become required based on rules |
| Field becomes hidden | DOM removal with `display: none` transition, then `unregister` or `setValue(undefined)` after 0ms timeout |
| Field becomes visible again | Field re-renders with default/empty value (previous value was cleared) |
| Field becomes conditionally required | Red asterisk appears next to label; Zod schema regenerated to enforce required |
| Calculation source value changes | Target calculation field value auto-updates in real-time |
| Calculation field in form | Rendered as readonly number input with `bg-gray-100` background and "자동 계산" badge |

### Section Interactions

| Interaction | Behavior |
|-------------|----------|
| Click section header (user form) | Toggle collapse/expand with 200ms animation |
| Section starts expanded | All sections default to expanded state on form load |
| Section hidden by condition | Section header and all child fields hidden; child field values cleared |
| Section in builder canvas | Rendered as FieldCard with left blue border accent; drag to reorder |
| Section in read-only mode | Section header shown without collapse toggle; all fields visible |

### Save with Circular Dependency

| Interaction | Behavior |
|-------------|----------|
| Click "저장" with circular dependency | Frontend runs detectCircularDeps; if cycle found, show red toast with cycle path; save is blocked |
| Click "저장" without circular dependency | Normal save proceeds; backend also validates (D-16) |
| Backend detects cycle (frontend missed) | API returns 400 with cycle path; frontend shows same red toast |

### Loading States

| Component | Loading Behavior |
|-----------|-----------------|
| Conditions tab | Instant render from builder state (no async loading) |
| Calculation fields | Instant computation (synchronous) |
| Section toggle | Instant collapse/expand (CSS transition only) |

### Error States

| Component | Error Behavior |
|-----------|---------------|
| Circular dependency | Red toast at top-center, 8s auto-dismiss, includes cycle path in monospace |
| Template save failure | Red toast with generic error message, 5s auto-dismiss |
| Invalid condition (deleted source field) | Source field dropdown shows "삭제된 필드" in red text; rule is visually flagged |

## Dark Mode

All new components must support dark mode via Tailwind `dark:` prefix, following established patterns:
- Backgrounds: `bg-white dark:bg-gray-900` (page), `bg-gray-50 dark:bg-gray-800` (cards, section headers)
- Borders: `border-gray-200 dark:border-gray-700`
- Text: `text-gray-900 dark:text-gray-50` (heading), `text-gray-600 dark:text-gray-400` (body)
- Interactive hover: `hover:bg-gray-100 dark:hover:bg-gray-700`
- Error: `text-red-700 dark:text-red-400`, `bg-red-50 dark:bg-red-900/30`
- Accent badges: `bg-blue-50 text-blue-700 dark:bg-blue-900/30 dark:text-blue-400`
