## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (mt-1 for error messages) |
| sm | 8px | Compact element spacing (gap-2 in table cells, px-2) |
| md | 16px | Default element spacing (space-y-4, px-4) |
| lg | 24px | Section padding (py-6 for form sections) |
| xl | 32px | Layout gaps (not used this phase) |
| 2xl | 48px | Major section breaks (not used this phase) |
| 3xl | 64px | Page-level spacing (not used this phase) |

Exceptions: Touch targets at 44px (h-11) for standard inputs, 36px (h-9) for table cell inputs, 32px (h-8 w-8) for icon-only delete buttons. All match existing hardcoded form patterns.

## Component Inventory

### DynamicForm (Edit Mode)

8 field type components, all within a single-column stack layout (D-07).

**Visual hierarchy:** First visible field is the primary visual anchor — auto-focused on mount, positioned at top of the single-column stack. Fields flow top-to-bottom in schema-defined order.

| Field Type | Input Control | Specs |
|------------|---------------|-------|
| text | `<input type="text">` | h-11, px-4, rounded-lg, border-gray-300, focus:ring-2 focus:ring-blue-500. Matches GeneralForm title input exactly. |
| textarea | `<textarea>` | w-full, min-h-[120px], px-4 py-3, rounded-lg, same border/focus as text. Resize vertical only. |
| number | `<input type="number">` | h-11, px-4, text-right, rounded-lg. Unit suffix rendered as inline text-sm text-gray-500 after input (if config.unit). |
| date | react-day-picker trigger button + popover | h-11, px-4, rounded-lg button showing selected date or placeholder. Calendar popover below. D-05 decision. |
| select | Headless UI Combobox | h-11, px-4, rounded-lg. Searchable dropdown. Options from schema snapshot. D-04 decision. |
| table | HTML table + useFieldArray | Bordered table with gray-50 header row. Cell inputs h-9 px-2 rounded text-sm. "+ 행 추가" button below. Row X delete button h-8 w-8. Matches ExpenseForm pattern exactly. D-06 decision. |
| staticText | `<div>` with rendered content | text-sm text-gray-700 dark:text-gray-300, py-2. No input, display-only informational text. |
| hidden | none (hidden from UI) | Rendered as `<input type="hidden">` with defaultValue from config. No visual presence. |

**Field wrapper pattern (each field):**
```
<div>                              // space-y-4 between fields
  <label class="text-sm font-semibold text-gray-700 dark:text-gray-300">
    {label} <span class="text-red-500">*</span>   // asterisk if required
  </label>
  {input component}
  <p class="mt-1 text-sm text-red-600">{error}</p>  // only when validation error
</div>
```

### DynamicReadOnly (Read-Only Mode)

Same field order as edit mode. Input controls replaced with text display (D-11).

| Field Type | Display |
|------------|---------|
| text | text-sm text-gray-900 dark:text-gray-50, plain text |
| textarea | text-sm text-gray-900, whitespace-pre-wrap |
| number | text-sm text-right, formatted number + unit suffix |
| date | text-sm, formatted date string (YYYY-MM-DD) |
| select | text-sm, resolved label from schema snapshot options (not raw value) |
| table | Read-only table, same header style, cell values as text (no inputs) |
| staticText | Same as edit mode |
| hidden | Not rendered |

**Empty field display:** Label + em dash "---" in text-sm text-gray-400 (D-12).

**Read-only field wrapper:**
```
<div>                              // space-y-4 between fields
  <label class="text-sm font-semibold text-gray-700 dark:text-gray-300">
    {label}
  </label>
  <div class="text-sm text-gray-900 dark:text-gray-50">
    {value || "---"}
  </div>
</div>
```

### TemplateSelectionModal (Extended)

Existing modal extended to show dynamic templates below hardcoded ones (D-03). Dynamic templates fetched from API. Same card/button pattern as existing items. No visual distinction between hardcoded and dynamic for end users.

## Interaction Contracts

### Form Validation (RNDR-03)

| Interaction | Behavior |
|-------------|----------|
| Blur field | Validate that field only (react-hook-form mode: "onBlur") |
| Submit form | Validate all fields, scroll to first error |
| Error display | Inline red text below the errored field (mt-1 text-sm text-red-600) |
| Error message language | Korean, generated from field label (D-09): "{label}을(를) 입력해주세요" for required, "{label}은(는) {min}자 이상이어야 합니다" for minLength |
| Zod schema generation | useMemo on mount, schemaToZod(fields) -> zodResolver (D-08) |

### Table Field (RNDR-04)

| Interaction | Behavior |
|-------------|----------|
| Add row | Click "+ 행 추가" button below table. New row appended with empty defaults. |
| Remove row | Click X button on row. Disabled if only minRows rows remain. |
| Row limit | maxRows from schema config. Hide add button when reached. |
| Cell validation | Per-cell Zod validation via nested z.object in z.array (D-10). Errors shown below the cell. |
| Empty table | Show table header + single empty row (matching minRows default of 1). |

### Select Field (Combobox)

| Interaction | Behavior |
|-------------|----------|
| Click | Open dropdown with all options visible |
| Type | Filter options by search text (case-insensitive Korean matching) |
| Select | Close dropdown, show selected label in trigger |
| Clear | Click X icon inside trigger to clear selection (if not required) |
| No results | Show "검색 결과가 없습니다" in dropdown |

### Date Field (Calendar Picker)

| Interaction | Behavior |
|-------------|----------|
| Click trigger | Open calendar popover below trigger button |
| Select date | Close popover, show formatted date (YYYY-MM-DD) in trigger |
| Clear | Click X icon inside trigger to clear date (if not required) |
| Keyboard | Arrow keys navigate calendar, Enter selects, Escape closes |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "임시저장" (Save Draft) -- existing, managed by DocumentEditorPage |
| Template badge | "{templateName}" in text-xs font-normal text-gray-500 |
| Empty field (read-only) | "---" (em dash, D-12) |
| Validation: required | "{label}을(를) 입력해주세요" |
| Validation: minLength | "{label}은(는) {min}자 이상이어야 합니다" |
| Validation: maxLength | "{label}은(는) {max}자 이하여야 합니다" |
| Validation: min (number) | "{label}은(는) {min} 이상이어야 합니다" |
| Validation: max (number) | "{label}은(는) {max} 이하여야 합니다" |
| Validation: table minRows | "최소 {min}개의 행이 필요합니다" |
| Validation: table maxRows | "최대 {max}개의 행까지 추가할 수 있습니다" |
| Table add row | "+ 행 추가" |
| Table remove row (aria) | "행 삭제" |
| Select no results | "검색 결과가 없습니다" |
| Select placeholder | "{label} 선택" |
| Date placeholder | "날짜 선택" |
| Error state (schema load fail) | "양식 스키마를 불러올 수 없습니다. 페이지를 새로고침해주세요." |
| Error state (no fields) | "이 템플릿에 정의된 필드가 없습니다." |

Destructive actions: Row deletion in table fields. No confirmation dialog needed (inline X button with undo possibility by re-adding). Matches existing ExpenseForm pattern.

## State Contracts

### DynamicForm States

| State | Visual |
|-------|--------|
| Loading schema | Skeleton placeholder: 3 gray-200 rounded-lg bars (h-11 w-full) with pulse animation |
| Schema loaded | Full form rendered with all field controls |
| Schema error | Error message card (border-red-200 bg-red-50 text-red-700, rounded-lg, p-4) with error copy |
| Empty schema (no fields) | Info message card (border-blue-200 bg-blue-50 text-blue-700, rounded-lg, p-4) |
| Validation errors | Inline error text per field, no top-level error banner |
| Submitting | Form inputs disabled, submit button shows spinner (existing pattern) |

### DynamicReadOnly States

| State | Visual |
|-------|--------|
| Loading | Same skeleton as edit mode |
| Loaded | Label-value pairs for all fields |
| Empty values | Label + "---" (D-12) |
| Schema snapshot missing | Fallback: render formData as JSON in `<pre>` code block with gray-100 bg |

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Field labels | All inputs linked via htmlFor/id. Label text from schema field.label. |
| Required indicator | aria-required="true" on required inputs. Visual asterisk in label. |
| Error messages | aria-describedby linking input to error paragraph. role="alert" on error text. |
| Combobox (select) | Headless UI provides built-in ARIA roles (combobox, listbox, option). |
| Calendar (date) | Keyboard navigable. aria-label on trigger button. |
| Table row delete | aria-label="행 삭제" on delete button. |
| Focus management | First field auto-focused on form mount. Focus moves to first error on submit. |
