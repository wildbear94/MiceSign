## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding (`px-1`) |
| sm | 8px | Compact element spacing (`px-2`, `py-2`), table cell padding |
| md | 16px | Default element spacing (`space-y-4`, `px-4`, `gap-4`) |
| lg | 24px | Section padding — not used in form templates |
| xl | 32px | Layout gaps — not used in form templates |

Exceptions: Touch target height is 44px (`h-11`) for all input fields and select dropdowns, matching existing template pattern. Table row inputs use 36px (`h-9`) for compact density.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `gray-900` (dark) | Form background, input background (`bg-white dark:bg-gray-900`) |
| Secondary (30%) | `gray-50` / `gray-800` (dark) | Table header/footer rows (`bg-gray-50 dark:bg-gray-800`) |
| Accent (10%) | `blue-500` / `blue-600` | Focus rings, auto-calculated display values, "add row" link text |
| Destructive | `red-500` / `red-600` | Validation error text, row delete hover state |

Accent reserved for:
- Input focus ring (`focus:ring-blue-500 focus:border-blue-500`)
- Auto-calculated read-only values (e.g., total amount, calculated hours) displayed as `text-blue-600`
- "Add row" action link text (`text-blue-600 hover:text-blue-700`)

Border colors:
- Input borders: `border-gray-300 dark:border-gray-600`
- Table borders: `border-gray-200 dark:border-gray-700`

Text colors:
- Primary text: `text-gray-900 dark:text-gray-50`
- Secondary text (labels): `text-gray-700 dark:text-gray-300`
- Tertiary text (table headers): `text-gray-600 dark:text-gray-400`
- Muted text (template badge, placeholders): `text-gray-500 dark:text-gray-400`
- Disabled/empty text: `text-gray-400`

Source: All values extracted from existing `ExpenseForm.tsx`, `LeaveForm.tsx` patterns. No new colors introduced.

## Interaction Contracts

### Editable Tables (PurchaseForm, BusinessTripForm)

| Interaction | Behavior |
|-------------|----------|
| Add row | Append row with empty defaults. Button below table with Plus icon. |
| Remove row | X button in last column. Disabled when only 1 row remains (`disabled:opacity-30 disabled:cursor-not-allowed`). |
| Auto-sum | Recalculate per-row amount and total on every keystroke via `watch('items')` + `useEffect`. |
| Min rows | At least 1 row required. Cannot remove the last row. |
| New row defaults | PurchaseForm: `{name:'', spec:'', quantity:1, unitPrice:0, amount:0}`. BusinessTrip itinerary: `{date:'', location:'', description:''}`. BusinessTrip expense: `{category:'transport', description:'', amount:0}`. |

### Auto-Calculations

| Form | Input | Output | Display |
|------|-------|--------|---------|
| PurchaseForm | quantity * unitPrice per row | Per-row amount + totalAmount | `formatCurrency(amount)` in table cell, `formatCurrency(total)` in footer |
| BusinessTripForm | Sum of all expense amounts | totalExpense | `formatCurrency(totalExpense)` in footer |
| OvertimeForm | endTime - startTime | hours (rounded to 0.5h) | `"X.X시간"` in `text-base font-semibold text-blue-600` |

### Overtime Time Calculation Edge Case

If `endTime < startTime` (overnight work, e.g., 22:00 to 02:00): add 24 hours to endMinutes before subtraction. Display calculated hours normally. Rounding: nearest 0.5 hours (30-minute increments).

### Select Dropdowns

| Form | Field | Options (hardcoded) |
|------|-------|---------------------|
| PurchaseForm | 결제 방법 | 법인카드, 계좌이체, 현금, 기타 |
| BusinessTripForm | 경비 분류 | 교통비, 숙박비, 식비, 기타 |

### Form Validation (Zod + React Hook Form)

Validation errors display as `<p className="mt-1 text-sm text-red-600">` immediately below the offending field. Real-time validation on blur/change + full validation on submit.

## ReadOnly Component Contract

Each form has a corresponding ReadOnly component that renders the same layout but with plain text instead of input fields. Rules:

1. Parse `formData` JSON string into the typed interface
2. If `formData` is null/undefined, render "내용 없음" in `text-sm text-gray-400`
3. Tables render with identical column structure but no edit controls (no add/remove buttons)
4. Currency values display with `formatCurrency()`
5. Auto-calculated values (hours, totals) display inline as plain text
6. No form element tags (no `<input>`, `<select>`, `<textarea>`)
