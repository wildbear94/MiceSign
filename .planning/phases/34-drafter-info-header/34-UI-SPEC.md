## Typography

| Role | Size | Tailwind Class | Weight | Line Height | Usage |
|------|------|----------------|--------|-------------|-------|

**Total declared sizes:** 2 (12px, 14px). **Total declared weights:** 2 (400, 500).

**Rationale:**
- 14px (`text-sm`) for values matches `DocumentDetailPage.tsx` L216 grid (`text-sm`) — header sits adjacent to the meta-info row and must visually align.
- 12px (`text-xs`) for labels matches the same grid's label pattern (project SoT: `<span className="text-gray-500 dark:text-gray-400 block mb-0.5">양식</span>`).
- `font-medium` (500) on values gives the drafter info subtle emphasis without competing with the form's title `text-base font-semibold` (`GeneralForm.tsx` L62).
- Empty/null `positionName` displays the literal placeholder `—` (em dash, U+2014) in the value slot at the same `text-sm font-medium` weight; cell is **not** hidden (preserves 4-column grid integrity).

**Font family:** All text inherits `font-sans` → Pretendard Variable from `tailwind.config.js`.

## Layout Contract

| Property | Specification |
|----------|---------------|
| Container element | `<dl>` (definition list — semantic for label/value pairs) |
| Outer wrapper class | `bg-white dark:bg-gray-900 border border-gray-200 dark:border-gray-700 rounded-xl p-4 mb-4` |
| Grid breakpoint | `grid grid-cols-2 md:grid-cols-4 gap-3` (Tailwind `md:` = ≥768px confirmed default) |
| Mobile (<768px) | 2 columns × 2 rows (2x2): row 1 = 부서 / 직위·직책, row 2 = 기안자 / 기안일 |
| Desktop (≥768px) | 4 columns × 1 row: 부서 / 직위·직책 / 기안자 / 기안일 |
| Cell internal structure | `<div>` containing `<dt class="text-xs text-gray-500 dark:text-gray-400 mb-0.5">{label}</dt>` + `<dd class="text-sm font-medium text-gray-900 dark:text-gray-50">{value}</dd>` |
| Vertical/horizontal dividers | **None** (minimalist — gap-3 + label hierarchy provide sufficient visual separation, matching the `DocumentDetailPage.tsx` L216 SoT) |
| Position in form | First child of `<form>` element (Edit) or first child of ReadOnly component root (`<div>` wrapping the read-only fields) — above the form's existing template label / title input |
| Margin below header | `mb-4` (16px) — visually separates header from form body |

**ARIA / Semantic structure:**

```html
<dl
  className="bg-white dark:bg-gray-900 border border-gray-200 dark:border-gray-700 rounded-xl p-4 mb-4 grid grid-cols-2 md:grid-cols-4 gap-3"
  aria-label={t('drafterInfo.headerAriaLabel')}
>
  <div>
    <dt className="text-xs text-gray-500 dark:text-gray-400 mb-0.5">{t('drafterInfo.departmentLabel')}</dt>
    <dd className="text-sm font-medium text-gray-900 dark:text-gray-50">{departmentName}</dd>
  </div>
  <div>
    <dt className="text-xs text-gray-500 dark:text-gray-400 mb-0.5">{t('drafterInfo.positionLabel')}</dt>
    <dd className="text-sm font-medium text-gray-900 dark:text-gray-50">{positionName ?? '—'}</dd>
  </div>
  <div>
    <dt className="text-xs text-gray-500 dark:text-gray-400 mb-0.5">{t('drafterInfo.drafterLabel')}</dt>
    <dd className="text-sm font-medium text-gray-900 dark:text-gray-50">
      {drafterName}
      {showLegacyBadge && (
        <span className="text-xs text-amber-600 dark:text-amber-400 ml-1">
          {t('drafterInfo.currentInfoBadge')}
        </span>
      )}
    </dd>
  </div>
  <div>
    <dt className="text-xs text-gray-500 dark:text-gray-400 mb-0.5">{t('drafterInfo.draftedAtLabel')}</dt>
    <dd className="text-sm font-medium text-gray-900 dark:text-gray-50">{formattedDate}</dd>
  </div>
</dl>
```

**Why `<dl>` (definition list):** The 4 fields are pure label→value pairs with no interactivity, no actions, and no relationship beyond "this label describes this value." `<dl>` is HTML's canonical structure for this; screen readers announce as "definition list with 4 terms" giving sensible reading order. Avoids the alternative (a `<div>` with `aria-label` only, which would force reliance on visual layout for grouping).

**Reading order (screen reader):** "기안자 정보 헤더 정의 목록" → "부서: 개발1팀" → "직위·직책: 팀장" → "기안자: 홍길동" → "기안일: 2026. 04. 29." (with "현재 정보" appended to drafter line if legacy fallback active).

## Date Format Contract

| Mode | Source | Format | Example |
|------|--------|--------|---------|
| DRAFT | (no value) | Literal `—` (em dash, U+2014) | `—` |
| SUBMITTED (snapshot) | `drafterSnapshot.draftedAt` (ISO `LocalDateTime`) | `new Date(s).toLocaleDateString('ko-KR', { year: 'numeric', month: '2-digit', day: '2-digit' })` | `2026. 04. 29.` |
| SUBMITTED (legacy fallback) | `existingDoc.submittedAt` (ISO) | Same as above | `2026. 04. 29.` |

**Rationale:** Project SoT is `DocumentDetailPage.tsx` L127~131 (`{ year:'numeric', month:'2-digit', day:'2-digit', hour:'2-digit', minute:'2-digit' }`). Header drops `hour`/`minute` because the field is "기안일" (date), not "기안 시각". Inline pattern (no central util) matches Pattern 6 of RESEARCH.md.

**Time zone:** Browser local (default `toLocaleDateString` behavior). Documents are submitted by the same user that views them in the immediate flow; cross-timezone display is not a v1.x concern.

## Component API Contract

```ts
// Location: frontend/src/features/document/components/DrafterInfoHeader.tsx (NEW)

export type DrafterSnapshot = {
  departmentName: string;
  positionName: string | null;
  drafterName: string;
  draftedAt: string; // ISO 8601 (LocalDateTime serialized)
};

export type DrafterLive = {
  drafterName: string;
  departmentName: string;
  positionName: string | null;
};

export type DrafterInfoHeaderProps =
  | { mode: 'draft'; live: DrafterLive }
  | {
      mode: 'submitted';
      snapshot: DrafterSnapshot | null; // null → legacy fallback
      live: DrafterLive;                 // always required (used when snapshot null)
      submittedAt: string;               // ISO — used only in legacy fallback path
    };
```

**Props selection rule (inside component):**
- `mode === 'draft'` → render `live.*` + date = `t('drafterInfo.draftedAtPlaceholder')` + `showLegacyBadge = false`
- `mode === 'submitted'` && `snapshot !== null` → render `snapshot.*` + formatted `snapshot.draftedAt` + `showLegacyBadge = false`
- `mode === 'submitted'` && `snapshot === null` → render `live.*` + formatted `submittedAt` + `showLegacyBadge = true`

**Rendering invariants:**
- Component is pure (no hooks beyond `useTranslation('document')`).
- No effects, no async, no error boundaries needed.
- Re-renders only when props change — parent caches via React.memo if list re-renders become a hot path (not expected in this phase; deferred).
