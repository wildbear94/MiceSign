## Interaction Contracts

### Template Activate/Deactivate (D-06)

| Step | Action | Result |
|------|--------|--------|
| 1 | Admin navigates to /admin/templates | Template list loads via `GET /api/v1/admin/templates` |
| 2 | Admin clicks toggle on active template | Confirmation dialog opens: "{{name}} 양식을 비활성화하시겠습니까?" |
| 3a | Admin confirms | `DELETE /api/v1/admin/templates/{id}` fires. Toggle flips off. Success toast "양식이 비활성화되었습니다". Query invalidated. |
| 3b | Admin cancels | Dialog closes, no change |
| 4 | Admin clicks toggle on inactive template | Confirmation dialog opens: "{{name}} 양식을 활성화하시겠습니까?" |
| 5a | Admin confirms | `PUT /api/v1/admin/templates/{id}` with `{ isActive: true }` fires. Toggle flips on. Success toast "양식이 활성화되었습니다". Query invalidated. |
| 5b | Admin cancels | Dialog closes, no change |

### Toggle State Machine

```
IDLE ──(click toggle)──> CONFIRMING (dialog)
                              |
                        (cancel)──> IDLE
                        (confirm)──> PROCESSING ──(success)──> IDLE (state flipped)
                                          |
                                    (error)──> IDLE + error toast
```

## Accessibility Notes

| Requirement | Implementation |
|-------------|----------------|
| Toggle switch | `role="switch"`, `aria-checked="true/false"`, `aria-label="{{name}} 양식 활성화"` |
| Toggle keyboard | Toggle on Enter or Space keypress |
| Confirmation dialog | Reuse `ConfirmDialog` — focus trap, ESC close, `role="dialog"`, `aria-modal="true"` |
| Table semantics | Use `<table>`, `<thead>`, `<tbody>`, `<th scope="col">` for proper screen reader navigation |
| Success/error toasts | `role="alert"` for screen reader announcement |
| Sidebar nav link | Existing `NavLink` pattern with `aria-current="page"` (handled by React Router) |
