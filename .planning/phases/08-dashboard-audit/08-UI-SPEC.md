## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding |
| sm | 8px | Compact element spacing, table cell vertical padding |
| md | 16px | Default element spacing, card internal padding |

| xl | 32px | Layout gaps between major sections (count row vs list row) |
| 2xl | 48px | Page top/bottom padding |

## Layout Contract

### DashboardPage Layout

```
+----------------------------------------------------------+
| 대시보드                              [새 문서 작성]       |  <- Header row, mb-8
+----------------------------------------------------------+
| [Count Card]    [Count Card]    [Count Card]              |  <- 3-column grid, gap-6
| 결재 대기 (N)    임시저장 (N)    완료 (N)                    |
+----------------------------------------------------------+
|                        gap-8                              |
+----------------------------+-----------------------------+
| 결재 대기 목록        더보기 | 최근 문서            더보기  |  <- 2-column grid, gap-6
| [table: 5 rows]            | [table: 5 rows]             |
|                            |                             |
+----------------------------+-----------------------------+
```

- Top row: 3 count cards in `grid grid-cols-3 gap-6`
- Bottom row: 2 list cards in `grid grid-cols-1 lg:grid-cols-2 gap-6`
- Each list card: white card with border, rounded-lg, p-6
- Responsive: On mobile (< lg), lists stack vertically

### CountCard Layout

```
+---------------------------+
|  [Icon]                   |  <- 40px icon in muted color
|  32                       |  <- Display size (28px), font-bold
|  결재 대기                  |  <- Label (12px), text-gray-500
+---------------------------+
```

- Card: `bg-white dark:bg-gray-800 border border-gray-200 dark:border-gray-700 rounded-lg p-6`
- Icon: 40px (`h-10 w-10`), color matches semantic meaning:
  - 결재 대기: `text-blue-500` with `Clock` icon
  - 임시저장: `text-gray-400` with `FileEdit` icon
  - 완료: `text-green-500` with `CheckCircle2` icon

### Navbar Badge (NavBadge)

```
결재 대기 [3]    <- Red circle overlaid on nav link text
```

- Parent link: `relative` positioning
- Badge: `absolute -top-1 -right-3 bg-red-500 text-white text-xs font-medium rounded-full h-5 min-w-[20px] flex items-center justify-center px-1`
- Hidden when count is 0 (do not render)
- Maximum display: "99+" for counts over 99

### AuditLogPage Layout

```
+----------------------------------------------------------+
| 감사 로그                                                  |  <- Heading
+----------------------------------------------------------+
| [Action Type ▼]  [User ▼]  [Start Date]  [End Date]  [검색] |  <- Filter bar
+----------------------------------------------------------+
| [Table: action, user, target, detail, IP, timestamp]      |  <- Paginated table
| ...                                                       |
+----------------------------------------------------------+
| [Pagination]                                              |
+----------------------------------------------------------+
```

- Filter bar: `flex flex-wrap gap-4 mb-6`
- Action type: `<select>` with all audit action options
- User: `<select>` or text input for user name search
- Date range: two `<input type="date">` fields
- Search button: `bg-blue-600 text-white px-4 py-2 rounded-lg`

## Interaction Contract

### Dashboard Interactions

| Interaction | Behavior |
|-------------|----------|
| Click count card | Navigate to respective full list page (결재 대기 -> /approvals/pending, 임시저장 -> /documents/my?status=DRAFT, 완료 -> /approvals/completed) |
| Click "더보기" link | Navigate to same target as count card click |
| Click pending list row | Navigate to `/documents/{documentId}` |
| Click recent documents row | Navigate to `/documents/{documentId}` |
| Click "새 문서 작성" | Open template selection modal (existing `TemplateSelectionModal`) |
| Auto-refresh | TanStack Query `refetchInterval: 60000` on summary + both lists |
| Tab focus resume | TanStack Query default: refetch on window focus |

### Audit Log Interactions

| Interaction | Behavior |
|-------------|----------|
| Change filter + click 검색 | Refetch audit logs with new filter parameters, reset to page 0 |
| Click pagination | Load page N of filtered results |
| No row click action | Audit log rows are read-only, no navigation |

### Loading States

| Component | Loading Behavior |
|-----------|-----------------|
| Count cards | Show pulse animation skeleton (`animate-pulse`) matching card dimensions |
| Dashboard lists | Show 3 skeleton rows matching existing pattern from PendingApprovalsPage |
| Audit log table | Show skeleton rows same as dashboard pattern |
| Navbar badge | Hidden until data loads (no flash of "0") |

### Error States

| Component | Error Behavior |
|-----------|---------------|
| Dashboard summary | Show "-" for count values, no error banner (silent degradation) |
| Dashboard lists | Show error message inline within card area |
| Audit log | Show error banner above table with retry suggestion |

## Accessibility Notes

| Element | Requirement |
|---------|-------------|
| Count cards | `role="link"` or wrap in `<a>` for keyboard navigation |
| Navbar badge | `aria-label="결재 대기 N건"` on the nav link when badge is shown |
| Table rows (clickable) | `role="button"` and `tabIndex={0}` with `onKeyDown` Enter handler |
| Filter controls | Associated `<label>` elements for each input |
| Loading skeletons | `aria-hidden="true"` on skeleton elements |

## Dark Mode

All components must support dark mode via Tailwind `dark:` prefix, following existing patterns:
- Backgrounds: `bg-white dark:bg-gray-900` (page), `bg-gray-50 dark:bg-gray-800` (cards)
- Borders: `border-gray-200 dark:border-gray-700`
- Text: `text-gray-900 dark:text-gray-50` (heading), `text-gray-600 dark:text-gray-400` (body)
- Table hover: `hover:bg-gray-50 dark:hover:bg-gray-800/50`
