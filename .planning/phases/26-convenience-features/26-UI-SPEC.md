# Phase 26 — UI Design Contract

> Visual and interaction contract for template convenience features (duplicate / JSON export / JSON import / preset gallery). Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla Tailwind) |
| Icon library | Lucide React ^1.7.0 |
| Font | Pretendard Variable (sans-serif fallback chain) |

Source: Consistent with Phase 22 / 23 / 24 / 25 UI-SPECs across the v1.1 milestone. No new dependencies.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (p-1, gap-1) | Icon button inner padding, inline icon gaps |
| sm | 8px (p-2, gap-2) | Icon button spacing between actions, badge padding |
| md | 16px (p-4, gap-4) | Modal body padding, choice card internal padding, preset card grid gap |
| lg | 24px (p-6, gap-6) | Choice modal body padding, preset gallery outer padding, error list container |
| xl | 32px (p-8, gap-8) | Preset gallery top/bottom breathing room |

Exceptions: none

| Role | Size | Weight | Line Height | Tailwind Class |

| Choice card title | 14px | 600 (semibold) | 1.5 | `text-sm font-semibold` |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `gray-800` (dark) | Modal background, choice card background, preset card background |
| Secondary (30%) | `gray-50` / `gray-900` (dark) | Choice modal body backdrop strip, preset gallery desk background, error list container background |
| Accent (10%) | `blue-600` / `blue-400` (dark) | Choice card hover border/ring, primary CTA buttons, Copy/Download icon hover, focus rings |
| Destructive | `red-500` / `red-400` (dark) | Import validation error text, error list bullet, destructive confirmation icon |
| Success | `emerald-600` / `emerald-400` (dark) | Import validation success badge (one-shot usage in ImportTemplateModal only) |

Accent reserved for:
- Choice card hover ring (`hover:ring-2 hover:ring-blue-600`)
- Primary CTA buttons inside ImportTemplateModal / TemplateCreateChoiceModal / PresetGallery ("가져오기", "선택")
- Copy/Download icon button hover color (`hover:text-blue-600`, matching existing Pencil pattern at TemplateTable.tsx:159)
- Focus rings on file input and preset card focus state
- NOT used on: close (X), cancel, reset buttons (these remain neutral `text-gray-*`).

Destructive reserved for:
- Import validation error text and path markers inside ImportTemplateModal
- NOT used elsewhere in this phase (no delete/withdraw actions).

Source: Inherits palette from Phase 22 / 23 UI-SPECs. Emerald is new but scoped to a single ephemeral success chip.

## State Machine (TemplateListPage)

```
closed ── (click "양식 추가") ──> choice
choice ── (select blank) ──> form (initialValues = undefined)
choice ── (select preset) ──> preset
choice ── (select import) ──> import
preset ── (select preset card) ──> form (initialValues = preset - prefix)
import ── (valid JSON + click 가져오기) ──> form (initialValues = parsed - prefix)
form   ── (save success OR cancel) ──> closed

(row click)
closed ── (click Copy icon) ──> form (initialValues = fetched detail + suffixed name - prefix)
closed ── (click Download icon) ──> (fetch detail → browser download, stays closed)
```

All transitions that land in `form` reuse a single `TemplateFormModal` instance with `initialValues` prop — ensuring identical create UX across entry points (D-04, D-10, D-17).

## Keyboard & Accessibility

| Interaction | Key | Behavior |
|-------------|-----|----------|
| Close any Phase 26 modal | `Escape` | Close current modal only; returns to previous state in the state machine (e.g. Preset `Esc` → Choice, Choice `Esc` → closed) |
| Activate choice card / preset card | `Enter` or `Space` | Same as click |
| Tab order | Tab / Shift+Tab | Focus trap within current modal (existing modal pattern) |
| File dropzone | `Enter` / `Space` on focus | Opens native file picker |
| Copy / Download row buttons | `Enter` | Same as click; existing keyboard behavior of Edit icon button |

Aria requirements:
- Copy button: `aria-label="{template.name} 복제"`
- Download button: `aria-label="{template.name} JSON 내보내기"`
- Choice cards: `role="button"` + `aria-label` matching the title
- Import error list: wrapped in `role="alert"` container so screen readers announce the first error when it appears
- Preset cards: `role="button"` + descriptive `aria-label`

## Validation Contract (Import flow)

| Rule | Trigger | Visual Feedback |
|------|---------|-----------------|
| File must be `.json` extension | On file select | Inline error below dropzone: `templates.importErrorNotJson` |
| File must be parseable JSON | On file select | Inline error below dropzone: `templates.importErrorNotJson` |
| `exportFormatVersion` must equal `1` | Zod parse | Top-of-list error: `templates.importErrorFormatVersion` |
| `templateImportSchema` passes | Zod parse | Success chip appears, "가져오기" button enables |
| `templateImportSchema` fails | Zod parse | Error container renders with grouped field paths (via Zod `.format()`) |
| prefix uniqueness | On save in TemplateFormModal | Inline error bubbled from backend 409 (D-11, existing message) |

Error list rendering: each error is one line — `<monospace path>: <message>`. Paths are normalized via `zodErrorsToList(error.format())` utility (to be added in the planner's plan). Up to 50 errors displayed; overflow count rendered as footer row.

## Export Flow (no UI surface beyond the click)

- Click `Download` icon → `onExport(template)` in TemplateListPage
- Flow: `templateApi.getDetail(id)` → `templateExport.serialize(detail)` → create `Blob([json], { type: 'application/json' })` → `URL.createObjectURL` → trigger `<a download={filename}>` → revoke URL
- Toast: success → `templates.exportSuccess`, failure → `templates.exportFailure`
- Filename: `{code}-{YYYYMMDD}.json` (D-06)
- No modal, no confirmation. One click = one file.
