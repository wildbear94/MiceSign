| Role | Size | Weight | Line Height | Tailwind Class |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `gray-800` (dark) | Column editor background, input backgrounds |
| Secondary (30%) | `gray-50` / `gray-700` (dark) | Column row collapsed header bg, preview table header bg |
| Accent (10%) | `blue-600` / `blue-400` (dark) | "컬럼 추가" button text, focus rings on column inputs |
| Destructive | `red-500` / `red-400` (dark) | Column delete button hover, required asterisk |
| Table identity | `teal-100`/`teal-700` / `teal-900/40`/`teal-300` (dark) | Column type badges (inherits table type color from constants.ts FIELD_TYPE_META) |

Accent reserved for:
- "컬럼 추가" add button text (`text-blue-600`)
- Focus ring on column label/config inputs (`focus:ring-blue-600`)
- NOT used on: drag handles, column collapse toggles, delete buttons (these use `gray-*` neutral styling)

Source: Existing FieldConfigEditor select-option pattern colors (FieldConfigEditor.tsx lines 164-170).

## Layout Contract

### Column Editor (inside FieldConfigEditor case 'table')

```
+----------------------------------------------------------+
| 컬럼 목록                                    [+ 컬럼 추가] |
+----------------------------------------------------------+
| +------------------------------------------------------+ |
| | [::] [ChevronRight] [TypeBadge] 컬럼 라벨      [삭제] | |  <- collapsed column row
| +------------------------------------------------------+ |
| +------------------------------------------------------+ |
| | [::] [ChevronDown]  [TypeBadge] 컬럼 라벨      [삭제] | |  <- expanded column row
| | +--------------------------------------------------+ | |
| | | 라벨: [________]    ID: [________]               | | |
| | | 타입: [select ▼]    필수: [checkbox]             | | |
| | | --- Type-specific config (ColumnConfigPanel) --- | | |
| | | text: placeholder, maxLength                     | | |
| | | number: min, max, unit                           | | |
| | | select: options list (add/remove)                | | |
| | +--------------------------------------------------+ | |
| +------------------------------------------------------+ |
+----------------------------------------------------------+
| 행 설정                                                   |
| 최소 행 수: [___]    최대 행 수: [___]                     |
+----------------------------------------------------------+
```

### Column Row Structure

Each column row follows the FieldCard collapse/expand pattern:
- **Collapsed:** Single line with drag handle, collapse chevron, type badge, label text, delete button
- **Expanded:** Below the header, bordered section with label/ID/type/required fields + type-specific config

Grid layout for column fields: `grid grid-cols-2 gap-3` (matching existing FieldConfigEditor pattern).

### Preview Table (in PreviewFieldRenderer)

```
+------------------------------------------------------+

| [disabled]   | [disabled]    | [disabled]             |  <- sample row 1
| [disabled]   | [disabled]    | [disabled]             |  <- sample row 2
+------------------------------------------------------+
| [+ 행 추가] (disabled, dashed border)                 |  <- add-row button (visual only)
```

## Drag & Drop Contract

| Property | Value |
|----------|-------|
| Library | @dnd-kit/core + @dnd-kit/sortable (D-07, D-08) |
| Drag handle | `GripVertical` icon, left side of column row |
| Drop indicator | Blue line between column rows (`border-blue-500`) |
| Drag overlay | Semi-transparent clone of column row (`opacity-80 shadow-lg`) |
| Activation | Pointer sensor with 8px distance threshold |
| Keyboard | `useSortable` keyboard sensor for accessibility |
| Constraint | Vertical axis only |

## Scroll Behavior

| Area | Scroll | Behavior |
|------|--------|----------|
| Column list container | No explicit scroll | Contained within FieldCard expanded area, which is inside left panel's `overflow-y-auto` |
| Column expanded config | No scroll | Expands the column row height inline |
| Preview table | `overflow-x-auto` | Horizontal scroll if many columns exceed panel width |

---

## Keyboard & Accessibility

| Interaction | Key | Behavior |
|-------------|-----|----------|
| Column reorder (keyboard) | Space to pick up, Arrow Up/Down to move, Space to drop | @dnd-kit keyboard sensor default behavior |
| Navigate between columns | Tab | Standard tab order through interactive elements |
| Delete column | Click Trash2 button | Immediate removal, no confirmation (D-03) |
| Expand/collapse column | Click row header or Enter when focused | Toggle expanded state |

All column inputs must have associated `<label>` elements or `aria-label` attributes.
Drag handle must have `aria-label="컬럼 순서 변경"` and `role="button"`.

## Validation Contract

| Rule | Trigger | Visual Feedback |
|------|---------|-----------------|
| Min 1 column on save | Form submission / save button | Toast or inline error: `templates.columnMinError` |
| Max 20 columns | "컬럼 추가" button click | Button disabled when 20 reached + tooltip with `templates.columnMaxError` |
| Column label required | On blur of empty label field | Red border on input (`border-red-500`) |
| minRows <= maxRows | On change of either field | Inline error text below field (`text-red-500 text-xs`) |
