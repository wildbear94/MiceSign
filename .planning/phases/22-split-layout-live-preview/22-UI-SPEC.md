## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `gray-800` (dark) | Modal background, form surfaces |
| Secondary (30%) | `gray-100` / `gray-900` (dark) | Preview panel background (paper desk), disabled input bg (`gray-50`/`gray-700`) |
| Accent (10%) | `blue-600` / `blue-400` (dark) | Active tab indicator, primary CTA button, focus rings |
| Destructive | `red-500` / `red-400` (dark) | Validation error text, required field asterisk |

Accent reserved for:
- Primary submit button (`bg-blue-600`)
- Active tab bottom border (`border-blue-600`)
- Focus ring on inputs (`focus:ring-blue-600`)
- NOT used on: preview toggle buttons, fullscreen button, close button (these use `gray-*` neutral styling)

Source: Existing TemplateFormModal.tsx color patterns (lines 163, 204, 324).

## Keyboard & Accessibility

| Interaction | Key | Behavior |
|-------------|-----|----------|
| Close modal | `Escape` | Close modal (existing behavior) |
| Close fullscreen | `Escape` | Close fullscreen only, stop propagation. Modal stays open. |
| Fullscreen close (X) | Click | Close fullscreen portal |
| Backdrop click (fullscreen) | Click | Close fullscreen portal |
| Backdrop click (modal) | Click | Close modal (existing behavior) |
| Focus trap | Tab/Shift+Tab | Existing modal focus trap. Fullscreen portal should trap focus independently. |

ESC priority: If fullscreen is open, ESC closes fullscreen only. The ESC handler must check `isFullscreen` state before calling `onClose`.
