    **방법 A (권장 — 최소 회귀):** PreviewFieldRenderer는 그대로 두되, 한 곳에서만 관리되도록 DynamicFieldRenderer의 `readOnly` 모드 + `value` prop을 활용. 즉 admin FormPreview는 **value + onChange 패턴**을 유지하되, DynamicFieldRenderer에 legacy prop 지원(`onChange`, `value`)을 추가하는 대신, PreviewFieldRenderer 파일 자체를 아래 구조로 재작성:

    ```tsx
    // PreviewFieldRenderer.tsx (thin wrapper)
    import type { SchemaField } from '../SchemaFieldEditor/types';
    import DynamicFieldRenderer from '../../../document/components/dynamic/DynamicFieldRenderer';
    import { adaptSchemaFieldToFieldDefinition } from '../../../document/components/dynamic/adaptSchemaField';
    import { useForm, FormProvider } from 'react-hook-form';
    import { useEffect } from 'react';

    interface PreviewFieldRendererProps {
      field: SchemaField;
      value: unknown;
      onChange: (value: unknown) => void;
      error?: string;
    }

    export default function PreviewFieldRenderer({ field, value, onChange, error }: PreviewFieldRendererProps) {
      const adapted = adaptSchemaFieldToFieldDefinition(field);
      const form = useForm({ defaultValues: { [field.id]: value } });

      // 상위 state와 양방향 동기화 (단일 필드 mini-form)
      useEffect(() => {
        const sub = form.watch((vals) => {
          const v = vals[field.id];
          if (v !== value) onChange(v);
        });
        return () => sub.unsubscribe();
      }, [form, field.id, value, onChange]);

      useEffect(() => {
        if (form.getValues(field.id) !== value) {
          form.setValue(field.id, value as any, { shouldDirty: false });
        }
      }, [value, field.id, form]);

      return (
        <FormProvider {...form}>
          <DynamicFieldRenderer
            field={adapted}
            mode="edit"
            register={form.register as any}
            control={form.control as any}
            error={error}
          />
        </FormProvider>
      );
    }
    ```

    **FormPreview.tsx 변경:** 기존 호출 방식(`<PreviewFieldRenderer field={f} value={formValues[f.id]} onChange={(v) => setFormValues(...)} />`)이 그대로 유지되는지 확인하고, 변경 불필요면 FormPreview.tsx는 손대지 않는다. 필요 시 import 경로만 수정.

    **중요:** FormPreview가 현재 PreviewFieldRenderer를 어떻게 호출하는지 **실행 전 반드시 읽고 prop 시그니처를 맞춘다**. 위 예시와 다르면 위 예시를 FormPreview 실제 사용 방식에 맞춰 조정한다. 예를 들어 FormPreview가 단일 formValues state 대신 다른 구조를 쓰면 그 구조에 맞춘다.

    **table 케이스 처리:** admin FormPreview의 table 렌더는 원본에서 `[0,1]`고정 disabled였다. DynamicFieldRenderer 위임 시 DynamicTableField가 editable 테이블을 보여주게 된다. 이는 admin 미리보기 UX 변경이므로:
    - admin FormPreview는 **editable table 허용**(미리보기니까 실제로 입력해볼 수 있음 — UX 개선)
    - 또는 DynamicFieldRenderer에 `tableMode: 'editable' | 'preview'` prop을 추가하여 preview에서는 2행 disabled 유지

    권장: **editable 허용** (더 나은 UX). 단 실행 시 admin에서 기존 2행 disabled UI 스크린샷과 비교하여 기능 회귀 없음을 확인한다. 회귀 발견 시 Plan 05에서 수동 체크리스트에 기록.

    **1) templateRegistry.ts** — 기존 registry는 건드리지 않고, fallback 헬퍼를 export:

    ```ts
    import DynamicCustomForm from '../dynamic/DynamicCustomForm';
    import DynamicCustomReadOnly from '../dynamic/DynamicCustomReadOnly';

    export const DYNAMIC_CUSTOM_EDIT = DynamicCustomForm;
    export const DYNAMIC_CUSTOM_READONLY = DynamicCustomReadOnly;

    /**
     * 지정된 templateCode가 하드코딩 TEMPLATE_REGISTRY에 없을 때
     * DynamicCustomForm/DynamicCustomReadOnly로 fallback 처리해야 함을 알린다.
     */
    export function isCustomTemplate(templateCode: string | undefined, hasSnapshot = false): boolean {
      if (hasSnapshot) return true;
      if (!templateCode) return false;
      return !(templateCode in TEMPLATE_REGISTRY);
    }
    ```

    (TEMPLATE_REGISTRY 원본 선언은 변경하지 않고, 새 헬퍼만 추가.)

    ```ts
    import { TEMPLATE_REGISTRY, DYNAMIC_CUSTOM_EDIT, isCustomTemplate } from '../components/templates/templateRegistry';

    const templateEntry = TEMPLATE_REGISTRY[resolvedTemplateCode];
    const schemaSnapshot = existingDoc?.schemaDefinitionSnapshot ?? null;
    const useCustomRenderer = !templateEntry && (
      isEditMode ? !!schemaSnapshot : isCustomTemplate(resolvedTemplateCode)
    );
    // 기존 "알 수 없는 양식" early return은 useCustomRenderer가 false일 때만:
    if (!templateEntry && !useCustomRenderer) {
      return <알 수 없는 양식>; // 기존 UI 유지

    const EditComponent = templateEntry?.editComponent ?? DYNAMIC_CUSTOM_EDIT;
    // EditComponent 렌더 시 initialData에 schemaSnapshot 전달:
    <EditComponent
      documentId={documentId}
      initialData={{
        title: existingDoc?.title ?? '',

      onSave={handleSave}
    />
    ```

    **중요:**
    - 실제 DocumentEditorPage의 코드 구조에 맞춰 변수명 조정 (`existingDoc`이 아닐 수 있음, `resolvedTemplateCode` 대신 `templateCode`일 수 있음)

    - 신규 작성 모드에서 `resolvedTemplateCode`가 URL로부터 오며 하드코딩 6종이 아니면 fallback
    - **라인별 비교 diff를 최소화**: 기존 return 흐름을 그대로 두고, 단지 `templateEntry` 존재 여부 분기 지점에 fallback만 삽입
