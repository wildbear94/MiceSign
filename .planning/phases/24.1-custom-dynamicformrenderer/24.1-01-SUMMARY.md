# Phase 24.1 Plan 01: 기반 타입 확장 및 검증 전략 확정 Summary

CUSTOM 템플릿을 위해 프론트엔드 문서 타입 계약에 schemaVersion/schemaDefinitionSnapshot 필드를 추가하고, TemplateEditProps/TemplateReadOnlyProps에 schemaSnapshot 전달 채널을 열었다. 인터페이스 우선(Interface-First) 원칙으로 downstream Wave가 타입 불일치 없이 병렬 진행 가능.

## What Was Built

### Task 1 — DocumentDetailResponse 확장 (commit `b47dcd2`)
`frontend/src/features/document/types/document.ts`의 `DocumentDetailResponse` 인터페이스에 2개 optional 필드 추가:

- `schemaVersion?: number | null` — 백엔드 `DocumentContent.schemaVersion`과 1:1 매핑
- `schemaDefinitionSnapshot?: string | null` — 백엔드가 자동 저장하는 JSON 직렬화 SchemaDefinition 스냅샷

JSDoc 주석으로 백엔드 필드 매핑과 D-10(백엔드 자동 저장) 출처를 명시. `CreateDocumentRequest`/`UpdateDocumentRequest`는 D-10에 따라 미변경.

검증: `npx tsc --noEmit` exit 0.
