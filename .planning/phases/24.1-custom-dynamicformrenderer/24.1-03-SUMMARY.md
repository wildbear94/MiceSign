# Phase 24.1 Plan 03: CUSTOM 양식 편집/조회 컴포넌트 Summary

CUSTOM 템플릿용 편집 컴포넌트(`DynamicCustomForm`)와 읽기 전용 컴포넌트(`DynamicCustomReadOnly`)를 구현했다. react-hook-form + 커스텀 Resolver + evaluateConditions + executeCalculations 조합으로 admin FormPreview 와 동일한 실시간 UX 를 제공하면서, D-16(숨김 필드 검증 제외), D-19(숨김 필드 값 보존), D-26(빈 행 자동 제거)을 동시에 만족한다. D-10 백엔드 자동 snapshot 정책에 따라 onSave payload 에 schema 는 포함되지 않는다.

## What Was Built

### Task 1a — DynamicCustomForm scaffold (commit `a3a21e3`)

`DynamicCustomForm.tsx` 신규 생성. 외부 컴포넌트는 schema 획득 책임을 진다:

- 신규 작성 모드(`existingSnapshotRaw === undefined`): `useQuery` + `templateApi.getTemplateSchema(templateCode)` 로 schema fetch (렌더링 전용, payload 미포함)
- 편집 모드: `initialData.schemaSnapshot` 을 try/catch 로 `JSON.parse`
- loading / error / invalid 각각의 안전한 fallback UI

내부 `DynamicFormInner` 컴포넌트:

- `parsedFormData = JSON.parse(initialData.formData)` (try/catch)
- `defaultValues = { __title, ...buildDefaultValues(schema), ...parsedFormData }`
- `useForm<FormValues>` + `zodResolver(schemaToZod(schema))` baseline (1b 에서 커스텀 resolver 로 교체)
- **`<FormProvider {...form}>` 로 감싸 자식 `DynamicTableField` 가 `useFormContext` 로 register 를 얻도록 함** (인계 사항 핵심)
- 제목 input + `schema.fields.map(...)` 으로 `DynamicFieldRenderer` 호출
- onSubmit 은 `JSON.stringify(formFields)` baseline (1c 에서 빈 행 제거 추가)

검증: `npx tsc --noEmit` exit 0.

### Task 1b — Custom Resolver + evaluateConditions (commit `427667d`)

- `hiddenFieldsRef = useRef<Set<string>>(new Set())` 추가
- 커스텀 `Resolver<FormValues>`: 매 검증 사이클마다 `visibleSchema = schema with required=false on hidden fields` 를 구성하고 `zodResolver(schemaToZod(visibleSchema))` 호출. closure 가 ref 를 참조하므로 ref 만 갱신해도 다음 사이클에 반영됨.
- `form.watch()` + `evaluateConditions(schema.conditionalRules ?? [], watched)` → `{ hiddenFields, requiredFields }` 매 렌더 계산
- `useEffect`로 `hiddenFieldsRef.current = hiddenFields` 동기화
- 렌더 루프 갱신: `isHidden = hiddenFields.has(field.id)` 판정
  - table 외 숨김 필드는 `return null` (값은 react-hook-form state 에 보존됨)
  - table 숨김 필드는 `<div style={{ display: 'none' }}>` 로 DOM 유지 → useFieldArray 의 row 상태 보존 (D-19)
- `dynamicRequired={requiredFields.has(field.id)}` 를 `DynamicFieldRenderer` 에 전달

검증: `npx tsc --noEmit` exit 0.

### Task 1c — executeCalculations + 빈 행 제거 (commit `7c57924`)

- `executeCalculations` import 추가
- `watchedKey = JSON.stringify(watched)` 를 의존성으로 하는 `useEffect`:
  - `executeCalculations(schema.calculationRules ?? [], watched)` 결과 순회
  - 각 결과 필드에 대해 `getValues(fid)` 와 비교, **다를 때만** `setValue(fid, v, { shouldValidate: false, shouldDirty: false })` 호출 → 무한 루프 방지
- `calcResultFieldIds = useMemo(...)` 로 계산 결과 필드 ID 집합 생성
- 렌더 루프의 `disabled` prop: `disabled={readOnly || calcResultFieldIds.has(field.id)}` (계산 결과 필드는 사용자 입력 차단)
- onSubmit 의 빈 행 제거 (D-26):
  ```ts
  for (const field of schema.fields) {
    const v = formFields[field.id];
    if (field.type === 'table' && Array.isArray(v)) {
      cleanedFields[field.id] = v.filter((row) =>
        Object.values(row).some(
          (cell) => cell !== '' && cell !== null && cell !== undefined,
        ),
      );
    } else {
      cleanedFields[field.id] = v;
    }
  }
  ```
  schema.fields 에 없는 키도 그대로 보존.
- onSave payload 는 여전히 `{ title, bodyHtml: undefined, formData: JSON.stringify(cleanedFields) }` 로 schema 미포함 (D-10)

### Task 2 — DynamicCustomReadOnly (commit `4b36aa4`)

`DynamicCustomReadOnly.tsx` 신규 생성:
