# Phase 24.1 — Validation Strategy

> Per-phase validation contract. **Manual-only test path** — 프로젝트에 테스트 러너(vitest)가 설치되어 있지 않으며, Phase 24와 동일하게 `tsc --noEmit` + `npm run build` + 수동 UAT 체크리스트로 검증한다. Nyquist 준수는 "모든 task가 `<automated>` verify를 가지며 sampling latency가 60초 이내"로 해석된다.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | **Manual-only** — vitest 미설치 (Phase 24 결정 유지). automated verify는 tsc + build로 한정. |
| **Config file** | none — test runner 설치하지 않음 |
| **Quick run command** | `cd frontend && npx tsc --noEmit` |
| **Full suite command** | `cd frontend && npm run build` + Plan 05 UAT 체크리스트 |
| **Estimated runtime** | tsc ~10초, build ~30–60초. 둘 다 60초 feedback latency 예산 내. |

---

## Sampling Rate

- **After every task commit:** `cd frontend && npx tsc --noEmit` (≤ 10초)
- **After every plan wave:** `cd frontend && npm run build` (≤ 60초)
- **Before `/gsd-verify-work`:** Full `npm run build` green + Plan 05 수동 UAT 체크리스트 전체 실행
- **Max feedback latency:** 60 seconds (build 기준)
- **Nyquist 간극 방지:** 각 task의 `<automated>`는 최소 `tsc --noEmit`을, 코드 생성 plan의 마지막 task는 추가로 `npm run build`를 포함한다. 3연속 task가 build 검증 없이 지나가지 않도록 Plan 03/04 말미에 build를 배치한다.
